# 超跌反弹策略 v4.0 升级方案 — 对比分析与数据可行性评估

> 编制日期：2026-04-22
> 当前版本：v3.3（oversold_bounce）
> 目标版本：v4.0（新量化规则）

---

## 一、全局架构对比

| 维度 | 当前 v3.3 | 新规则 v4.0 | 差异说明 |
|------|-----------|-------------|---------|
| **总分结构** | 6维度，理论105分硬截断100 | 6维度+3额外加分，理论115分硬截断100 | 新规则增加3个额外加分项 |
| **前置过滤** | 4项（ST/上市90天/市值/近20日跌幅） | 5项（+成交额过滤+创新低硬过滤+MA250超跌定义） | 过滤更严格，减少噪音 |
| **数据周期** | 近20日/40日为主 | 引入近60日、250日、MA250 | 长周期锚点更稳定 |
| **评分精细度** | 粗粒度档位（3-4档） | 细粒度档位（4-5档）+ 额外加分 | 评分更灵敏 |

## 二、逐条对比分析

### （一）前置过滤

| 过滤项 | 当前 v3.3 | 新规则 v4.0 | 差异评估 |
|--------|-----------|-------------|---------|
| **上市天数** | ≥90个交易日 | **≥120个交易日** | ⬆️ 更严格，规避次新股波动 |
| **ST/退市** | 名称匹配排除ST/*ST | 一致 | — |
| **日均成交额** | ❌ 无此过滤 | **近20日均成交额≥5000万元** | ⬆️ 新增硬过滤，保证流动性 |
| **创新低剔除** | 在HL结构中-5分惩罚 | **硬过滤：近20日最低>近60日最低×1.01** | ⬆️ 从惩罚改为前置过滤，更干净 |
| **超跌定义** | 近20日跌幅 > -20%（参数化） | **收盘价/MA250 ≤ 0.75** | ⬆️ 从短期跌幅改为年线相对位置，更稳健 |

**数据可行性：** ✅ 全部可行
- 上市天数：`stock_basic.list_date`
- 成交额：`daily.amount`（千元）→ 20日累加/20
- 创新低：`daily.low` → 近20日min vs 近60日min
- MA250：`daily.close` → 计算250日SMA

### （二）趋势维度（40分）

#### 2.1 跌幅深度（20分）

| 档位 | 当前 v3.3（近20日跌幅） | 新规则 v4.0（收盘价/MA250） | 评估 |
|------|------------------------|---------------------------|------|
| 20分 | < -50% | D ≤ -50% | ✅ 数据等价，均可计算 |
| 17分 | -50%~-40% | -50% < D ≤ -40% | ✅ |
| 13分 | -40%~-30% | -40% < D ≤ -30% | ✅ |
| 10分 | -30%~-25% | -30% < D ≤ -25% | ✅ |
| 7分 | -25%~-20% | ❌ 无此档（已在过滤阶段排除） | ⚠️ 新规则过滤门槛更严，-25%以下才进入评分 |

**关键差异：** 当前用"近20日跌幅"，新规则用"收盘价/MA250"。
- **近20日跌幅**：短期波动敏感，易受短期情绪影响
- **MA250相对位置**：以年线为锚，过滤掉短期假性超跌

**数据可行性：** ✅ `daily.close` 获取250日数据计算 MA250

**数据可行性：** ✅ 需要近60日数据，`daily.close` 可计算

#### 2.3 三重止跌信号（15分）

| 信号 | 当前 v3.3 | 新规则 v4.0 | 可行性 |
|------|-----------|-------------|--------|
| **信号A** | 长下影线（标准K线公式） | **低点不破：近3日最低价均 > 近5日最低价** | ✅ `daily.low` 可计算 |
| **信号B** | 放量阳线（涨幅>1% + 量比>阈值） | **RSI(14)从<30回升至≥35（近3日内发生）** | ⚠️ **需手动计算RSI** |
| **信号C** | MACD金叉（DIF上穿DEA） | **缩量后放量：前一日量比<0.7，当日量比≥1.2** | ✅ `daily.vol` 可计算 |

**⚠️ RSI 数据来源问题：**
- Tushare 无直接 RSI 接口
- 需用 `daily.close` 手动计算：
  ```
  RSI(14) = 100 - 100/(1 + RS)
  RS = 14日平均上涨幅度 / 14日平均下跌幅度
  ```
- **建议：** 在 `helpers.py` 新增 `calc_rsi()` 函数，计算成本极低

### （四）资金流向维度（15分 + 5分额外）

#### 4.1 当前 v3.3 资金流向

- 主力净流入额 > 0 → +10分
- 连续净流入天数 ≥ 3 → +5分
- 流入占比 > 1% → +5分（硬上限15）

**使用接口：** `moneyflow.net_mf_amount`（主力净流入）

#### 4.2 新规则 v4.0 资金流向

| 子项 | 规则 | 分值 | 数据来源 | 可行性 |
|------|------|------|---------|--------|
| 大单净流入方向 | 当日大单净流入额 > 0 | +5 | `moneyflow` lg/elg | ✅ |
| 大单占比 | 大单净流入额 / 当日成交额 ≥ 2% | +5 | `moneyflow` + `daily.amount` | ✅ |
| 大单排名 | 大单占比在全市场前20% | +5 | 需全市场排序 | ⚠️ |
| **资金逆转**（额外） | 前1日大单净流入<0 且 当日>0 | +5 | 需2日数据 | ✅ |

**大单定义：** `moneyflow` 提供 `lg`（大单）和 `elg`（特大单）：
```
大单净流入 = (buy_lg_amount + buy_elg_amount) - (sell_lg_amount + sell_elg_amount)
```

**⚠️ 大单排名问题：**
- 需要计算全市场所有股票的大单占比，排序取前20%
- 计算量：5000+只股票，每日运行一次
- **可行但性能需注意：** 可复用批量获取的 `moneyflow` 数据，在内存中排序

### （六）基本面加分项（10分）

| 子项 | 当前 v3.3 | 新规则 v4.0 | 可行性 |

**⚠️ 业绩预增数据源问题：**

## 四、实施建议

### 4.1 分阶段实施路线

```
Phase A（立即可做，1-2天）
├── 前置过滤：上市120天、成交额5000万、创新低硬过滤、MA250超跌
├── 评分：急跌加成、均线站稳、均线交叉（新增MA10）、有效阳线2%
├── 评分：HL结构细分（低点/高点抬高各5分）
├── 评分：市值弹性固定[50,200]
└── 前置过滤：max_circ_mv=0 表示不限（已在前一commit完成）

Phase B（需要开发，2-3天）
├── RSI(14)计算函数 + 三重止跌信号改造
├── 大单数据细化（方向/占比/排名）
├── 资金逆转信号（需存储前日数据）
├── 概念拥挤度惩罚
└── 概念成交额排名（vol降级实现）

Phase C（可选增强，1-2天）
├── 业绩预增改为净利润增速>30%（fina_indicator）
├── 大单排名性能优化（缓存全市场排序结果）
└── 前端参数滑块同步更新
```

### 4.2 关键决策点

| 决策点 | 建议方案 | 理由 |
|--------|---------|------|
| 概念成交额无amount字段 | **用vol（成交量）代替** | 方向一致，实现简单 |
| 大单排名计算量大 | **复用batch_moneyflow缓存，内存排序** | 已有批量获取机制 |
| RSI手动计算 | **helpers.py新增calc_rsi** | 14行代码，零依赖 |
| 净利润增速滞后 | **保留forecast为主，fina_indicator为辅** | 保证时效性 |
| 总分溢出 | **额外加分项不占用维度上限，总分min(100)** | 与当前一致 |

### 4.3 风险提示

1. **MA250计算需要250日历史数据** → 新股/次新股可能数据不足，已在上市天数过滤中规避
2. **全市场大单排名性能** → 5000+只股票排序，建议在批量获取moneyflow后统一计算
3. **RSI需要14+1=15日数据** → 已在daily获取中覆盖（当前获取45日）
4. **fina_indicator数据滞后** → 季报数据，可能无法反映最新业绩变化

## 五、结论

**总体评估：新规则可行，建议实施。**

| 评估维度 | 评分 | 说明 |
|---------|------|------|
| 数据可获得性 | 8.5/10 | 90%指标可直接获取，仅概念成交额需降级 |
| 计算复杂度 | 7/10 | RSI、大单排名新增计算量，但可控 |
| 策略逻辑严密性 | 9/10 | 前置过滤更严格、评分更精细、长周期锚点更稳健 |
| 与当前系统兼容性 | 8/10 | 复用现有批量数据获取框架，改动集中在screener.py |

**核心优势：**
1. **MA250锚点**比近20日跌幅更稳健，避免短期噪音
2. **成交额硬过滤**直接剔除低流动性标的，减少闪崩风险
3. **RSI回升信号**识别超卖反弹，比单纯MACD金叉更敏感
4. **大单数据**比笼统的"主力净流入"更精准

**建议决策：** 分Phase A→B→C渐进实施，Phase A可立即开始。
