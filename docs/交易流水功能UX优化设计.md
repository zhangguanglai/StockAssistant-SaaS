# 交易流水功能 UX 优化设计文档

## 一、现状问题分析

### 1.1 信息架构问题

| 问题 | 描述 | 影响 |
|------|------|------|
| **功能定位模糊** | "交易流水" Tab 同时包含"资金管理"和"交易记录"两个不相关的功能 | 用户认知负担重，找功能困难 |
| **数据层级混乱** | 资金管理（静态配置）→ 统计卡片（动态汇总）→ 交易表格（明细流水）混合展示 | 信息优先级不清晰 |
| **缺少时间维度** | 流水表格按时间倒序排列，但没有日期分组或筛选 | 大量记录时难以定位 |

### 1.2 交互体验问题

| 问题 | 描述 | 影响 |
|------|------|------|
| **资金管理入口深** | 修改初始资金需要进入交易流水Tab，不符合心智模型 | 新用户难以发现 |
| **表格信息过载** | 10列数据平铺，买入理由/情绪/备注列宽受限显示不全 | 关键信息被截断 |
| **无筛选能力** | 无法按股票、类型、时间范围筛选 | 查找特定记录困难 |
| **缺少导出** | 交易记录无法导出备份 | 数据安全感不足 |

### 1.3 视觉呈现问题

| 问题 | 描述 | 影响 |
|------|------|------|
| **统计卡片与表格粘连** | 无间距分隔，视觉层次混乱 | 阅读疲劳 |
| **盈亏列显示不一致** | 买入显示"-"，卖出显示金额，对齐方式不同 | 扫描对比困难 |
| **缺少空状态引导** | 无记录时仅显示"暂无交易记录" | 未引导用户进行首笔交易 |

## 二、优化设计方案

### 2.1 信息架构重组

```
【原结构】                    【新结构】
交易流水 Tab                  ├─ 持仓 Tab（新增资金管理入口）
├─ 资金管理（静态配置）   →   │   └─ 资金卡片 + 设置按钮
├─ 统计卡片（5个指标）        │
└─ 交易表格（10列）           ├─ 交易记录 Tab（专注流水）
                              │   ├─ 筛选工具栏
                              │   ├─ 时间线视图
                              │   └─ 交易详情抽屉
                              │
                              └─ 设置 → 资金管理弹窗
```

### 2.2 具体优化方案

#### A. 资金管理迁移（高优先级）

**现状**：资金管理藏在交易流水Tab内
**优化**：
1. 在「持仓」Tab 的资产摘要卡片增加「💰 资金设置」按钮
2. 点击弹出「资金管理」抽屉/弹窗
3. 原交易流水Tab内的资金管理区块移除

**理由**：
- 资金管理是账户级配置，与持仓资产直接相关
- 减少交易流水Tab的信息负担
- 符合"设置就近原则"

#### B. 交易流水界面重构（高优先级）

**新布局**：
```
┌─────────────────────────────────────────────────────────┐
│  📋 交易记录                                    [导出📤] │
├─────────────────────────────────────────────────────────┤
│  🔍 搜索股票 │ [全部类型▼] │ [本月▼] │ [重置筛选]       │
├─────────────────────────────────────────────────────────┤
│  ┌─────────────────────────────────────────────────┐   │
│  │ 📅 2026年4月                                    │   │
│  │ ┌─────────────────────────────────────────────┐ │   │
│  │ │ 🟢 买入  平安银行  ¥10.50 × 1000股  -¥10,510 │ │   │
│  │ │    理由：趋势突破 │ 情绪：自信 │ 04-09 10:30   │ │   │
│  │ └─────────────────────────────────────────────┘ │   │
│  │ ┌─────────────────────────────────────────────┐ │   │
│  │ │ 🔴 卖出  招商银行  ¥45.20 × 500股   +¥1,250  │ │   │
│  │ │    盈亏：+5.2% │ 情绪：贪婪 │ 04-08 14:15   │ │   │
│  │ └─────────────────────────────────────────────┘ │   │
│  └─────────────────────────────────────────────────┘   │
│  ┌─────────────────────────────────────────────────┐   │
│  │ 📅 2026年3月                                    │   │
│  │ ...                                             │   │
│  └─────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────┘
```

**关键改进**：
1. **时间线分组**：按月份/日期分组，便于追溯
2. **卡片式记录**：替代表格，信息展示更灵活
3. **类型标签色块**：买入绿色、卖出红色（A股习惯）
4. **盈亏可视化**：卖出记录显示盈亏百分比色块

#### C. 筛选与搜索能力（中优先级）

**新增筛选器**：
- **股票搜索**：输入代码/名称实时过滤
- **类型筛选**：全部 / 买入 / 卖出
- **时间范围**：本月 / 本季 / 本年 / 自定义
- **盈亏筛选**：全部 / 盈利 / 亏损（仅卖出）

#### D. 交易详情抽屉（中优先级）

**交互**：点击交易记录 → 右侧滑出详情抽屉
**内容**：
```

│ 平安银行 000001.SZ  │
│ 🟢 买入  04-09 10:30 │
├─────────────────────┤

│ 趋势突破，MA20上穿  │
│                     │

│                     │

│ 观察3天后加仓       │
└─────────────────────┘
```

#### E. 统计卡片优化（低优先级）

**现状**：5个卡片平铺（买入笔数/卖出笔数/胜率/手续费/已实现盈亏）
**优化**：
- 精简为3个核心指标：交易次数 / 胜率 / 累计盈亏
- 增加「查看完整分析」链接 → 跳转复盘Tab
- 卡片悬浮显示tooltip说明计算口径

#### F. 空状态优化（低优先级）

**现状**：仅显示"暂无交易记录"
**优化**：
```
┌─────────────────────────┐
│         📋              │
│    暂无交易记录          │
│                         │
│  在持仓页点击「买入」    │
│  记录您的第一笔交易      │
│                         │
│    [前往持仓页]          │
└─────────────────────────┘
```

## 四、界面原型要点

### 4.1 颜色规范
- 买入标签：`--green` (#10b981) 背景，白色文字
- 卖出标签：`--red` (#ef4444) 背景，白色文字
- 盈利金额：`--red` (#ef4444) - A股红涨绿跌
- 亏损金额：`--green` (#10b981)

### 4.2 间距规范
- 时间分组标题：margin-top: 24px
- 记录卡片间距：gap: 12px
- 卡片内边距：padding: 16px
- 筛选工具栏与列表：margin-bottom: 16px

### 4.3 交互细节
- 记录卡片hover：背景色加深，显示「查看详情」提示
- 筛选条件变化：列表即时刷新，无刷新按钮
- 时间分组：默认展开最近月份，其他月份可折叠

---

## 五、数据与性能考虑

1. **分页加载**：交易记录超过50条时分页，避免一次性渲染过多
2. **本地缓存**：筛选条件保存到localStorage，下次访问自动恢复
3. **导出格式**：支持CSV（Excel兼容）和JSON两种格式
