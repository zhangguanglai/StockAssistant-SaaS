## 二、当前筛选流程（6步漏斗）

### 第1步：大盘环境风控
```python
# screener.py L1691
if market["status"] == "下降" and not force:
    → 直接返回空结果，message="大盘处于下降趋势，建议观望"
```
**⚠️ 风险点**：这是第一道也是最狠的关卡。如果大盘被判定为"下降"，用户不勾选"强制模式"就永远无法运行。

### 第2步：基础过滤（basic_filter, L913-1057）
| 过滤条件 | 默认阈值 | 影响 |
|---------|---------|------|
| 排除ST股 | stock_st API精确匹配 | 排除~100只 |
| 排除停牌 | suspend_d API | 排除~50只 |
| 排除新股 | 上市<60天 | 排除~30只 |
| 流通市值下限 | ≥50亿 | 🔴 **大量排除中小盘** |
| 流通市值上限 | ≤300亿 | 排除超大蓝筹 |
| 换手率 | ≥3% | 🔴 **排除大部分蓝筹/白马** |
| 排除涨停 | 涨幅>9.5% | 少量 |

### 第3步：趋势确认（trend_confirm, L1157-1290）
**6个强制条件，全部AND关系：**

| # | 条件 | 默认阈值 | 严格度 |
|---|------|---------|-------|
| C1 | 收盘价 > MA20×(1+偏离%) | 偏离≥1% | ⭐⭐ 中等 |
| C2 | MA20连续3日向上 | — | ⭐⭐ 中等 |
| C3 | **MACD金叉（强制）** | DIF穿越DEA 或 柱状图由负转正 | 🔴🔴🔴 **极严** |
| C4 | 量比 ≥ 阈值 | ≥1.5 | ⭐⭐ 中等 |
| C5 | 当日涨幅 ≥ 阈值 | ≥2% | ⭐⭐ 中等 |
| C6 | **三看检查全通过（强制）** | 高低点抬高+均线多头+量价配合 | 🔴🔴🔴 **极严** |

### 第4步：板块与资金（sector_and_money_filter, L1297-1429）
| 条件 | 正常模式 | 降级模式 |
|------|---------|---------|
| 板块涨幅 > 阈值 | 必须 > 0.5% | 属于热门政策概念即可 |
| 主力净流入 > 0 | ✅ 必须正流入 | ❌ 不要求 |
| 流入天数 ≥ N天 | 必须≥2天 | ❌ 不要求 |

### 第5步：加分项评分（bonus_scoring, L1436-1572）
非过滤项，只影响排序。最高可加：涨停基因(10) + MACD金叉(10) + 缩量回踩(10) + 情绪(5) + 政策概念(8) + 业绩预告(8) + 回购(5) = **56分（但实际capped在合理范围）**

### 第6步：汇总排序（final_ranking, L1579-1624）
按总分 = trend_score + sector_score + money_score + bonus_score 排序

## 三、评分体系对照审查

### PRD定义 vs 实际实现

| 维度 | PRD定义范围 | 实际实现范围 | 一致性 |
|------|-----------|------------|--------|
| 趋势分(trend_score) | 0-40 | 0-40（偏离8 + 斜率12 + HL结构≤10 + 其他≤10） | ✅ |
| 板块分(sector_score) | 0-20 | 0-20（正常12+8=20 / 降级固定5分） | ✅ |
| 资金分(money_score) | 0-20 | 0-20（流入强度10 + 天数10） | ✅ |
| 加分(bonus_score) | 0-20 | 0-56（无上限限制！） | ⚠️ **超上限** |
| **总分** | **0-100** | **0-136** | ⚠️ **超出PRD定义** |

**发现1：bonus_score 无上限约束**
- PRD定义加分项上限20分，但实现中 bonus_score 可达56分
- 这导致总分可能超过100分，破坏了评分体系的参照意义
- 建议：bonus_scoring 结束后加 `c["bonus_score"] = min(c["bonus_score"], 20)`

## 四、🔴 致命问题分析：为什么一直筛不出结果？

### 问题1：MACD金叉是"瞬时事件"，不是"持续状态"

**现状**（L1196-1214）：
```python
# 金叉判断：DIF从下方穿越DEA，或MACD柱由负转正
is_cross = (dif_prev <= dea_prev) and (dif_now > dea_prev)      # DIF穿DEA
is_hist_cross = (macd_prev <= 0) and (macd_now > 0)             # 柱状图转正

if is_cross or is_hist_cross:
    macd_valid = True
```

**问题本质**：
- 金叉是一个**瞬间发生的事件**（某一天DIF刚好从下方穿过DEA）
- 大部分时间股票处于"已金叉后多头排列"状态，但**不再满足"正在金叉"的条件**
- 这意味着策略只能抓到**金叉发生的那一天**的股票，错过所有已经完成金叉但仍强势的标的
- 在一个5000+只的市场里，可能只有个位数股票在某一天恰好发生金叉

**专家建议**：
```
改为"MACD多头格局"判断：
✓ DIF > DEA（已经在上方，维持多头）
✓ DIF > 0 且 DEA > 0（零轴上方运作，更强）
✅ 可选：最近5日内发生过金叉（给额外加分）
```

### 问题2：三看检查过于严苛（三重AND）

**现状**（L1064-1150），三看全部必须通过：

| 三看 | 条件 | 通过率估计 |
|------|------|-----------|
| 一看·高低点 | 低点抬高 + 高点抬高同时满足 | ~25-35% |
| 二看·均线 | **MA5 > MA10 > MA20 严格多头** | ~15-25% |
| 三看·量价 | 量比>1.0 且 上涨日量>下跌日量 | ~30-40% |

**三重AND综合通过率** ≈ 25% × 20% × 35% = **1.75%**

也就是说，即使通过了前面5个技术条件，到这里还要再砍掉98%。结合前面的漏斗，最终通过率可能低于万分之一。

**专家建议**：
```
三看改为"加权评分制"而非"一票否决制"：
- 三看全过：+10分（最强信号）
- 两看通过：+5分
- 一看通过：+2分
- 允许带弱项通过，但在评分中体现
```

### 问题3：MA20连续3日向上 + 当日涨幅≥2% 的组合矛盾

**逻辑冲突**：
- MA20连续3日向上说明趋势稳健（通常是缓涨或震荡上行）
- 但当日涨幅≥2%又要求当日有较大突破
- 这两个条件组合在一起，实际上是在找"**温和趋势中突然放量突破**"的极端形态
- 这种形态在市场中的自然出现频率很低

**数据估算**：
- MA20连续向上：约30-40%的股票满足
- 当日涨幅≥2%：约15-20%的股票满足（正常交易日）
- 组合概率：≈ 6%

### 问题4：换手率≥3% + 流通市值50-300亿的组合偏窄

这个参数组合排除了：
- 大蓝筹（茅台、招商银行等换手通常<1%）→ 被市值和换手双杀
- 小盘妖股（流通市值<50亿）→ 被市值排除
- 中盘绩优股（换手1-2%，如很多制造业龙头）→ 被换手排除

**实际保留的池子**：主要是中盘活跃股（50-300亿，日换手3%+），大约占全市场的 **15-20%**

### 问题5：板块预过滤进一步收窄

**L1774-1823**：先确定热门板块成分股，再筛选。
- 如果 `get_hot_board_constituents()` 返回的热门板块成分股集合较小
- 或者该函数因接口问题返回空
- 则预过滤后的候选池大幅缩减

### 问题6：资金净流入>0 的硬性要求

**L1391-1394**（正常模式）：
```python
if total_net_in <= 0:
    continue          # ← 净流出直接淘汰
if inflow_days < p_min_inflow_days:
    continue          # ← 流入天数不够淘汰
```

- Tushare的 `moneyflow` 数据基于大单统计，有一定误差
- 很多优质突破票当天主力资金可能是小幅净流出的（主力边洗边拉）
- "近5日必须有≥2天净流入"的要求在震荡市中较难满足

## 五、量化估算：漏斗收敛模型

以全市场5200只为起点：

| 步骤 | 过滤条件 | 预估剩余 | 累计淘汰率 |
|------|---------|---------|-----------|
| 起点 | 全A股 | 5,200 | — |

| 第3步-C2 | MA20连升3日 | 60-120 | 97-98% |

## 六、优化方案

### 方案A：保守优化（保持原逻辑框架，放宽关键阈值）

| 参数 | 当前值 | 建议值 | 理由 |
|------|-------|--------|------|
| MACD条件 | 当天必须金叉 | **DIF>DEA（多头排列即可）** | 金叉是瞬时事件，多头是持续性状态 |
| 三看检查 | 强制全过 | **改为至少通过2看** | 降低误杀率 |
| 最小换手率 | 3% | **1.5%** | 覆盖更多中盘标的 |
| MA20偏离度 | ≥1% | **≥0%（站上即算）** | 降低门槛 |
| 最小涨幅 | ≥2% | **≥0.5%** | 允许温和突破 |
| 量比 | ≥1.5 | **≥1.2** | 适度放量即可 |

### 方案B：架构优化（推荐✅）

将趋势确认从"硬性过滤器"改为**"分层评分制"**：

#### 新设计：趋势突破 v2.0

**核心思路**：取消硬性淘汰，改用加权打分，用分数自然排序

| 维度 | 子项 | 满分 | 评分标准 |
|------|------|------|---------|
| **趋势面(40分)** | MA20位置 | 10 | 站上MA20得6分，偏离1-3%得8分，偏离>3%得4分（太远追高扣分） |
| | MA20方向 | 10 | 连续3日向上满分，2日向上7分，走平4分，微降1分 |
| | **MACD状态** | 12 | **DIF>DEA且>0轴 得12分；仅DIF>DEA 得8分；刚金叉额外+3分** |
| | 高低点结构 | 8 | HL双抬满分，单抬半分，震荡0分 |
| **板块面(20分)** | 所属板块涨幅 | 12 | >3%满分，>2%得10分，>0.5%得7分 |
| | 板块热度排名 | 8 | 前10概念得8分，前20得5分 |
| **资金面(20分)** | 流入强度 | 10 | 流入占比>3%得10分，>1%得8分，>0得5分 |
| | 流入持续性 | 10 | 5天全流入满分，4天8分，≥2天5分，允许净出流但扣分 |
| **加分项(20分)** | 三看检查 | 10 | **三看过全得10分，两过得6分，一过得3分** |
| | 涨停基因/缩量回踩/业绩/回购 | 10 | 各项累加（capped at 10） |

**关键变化**：
1. MACD从"硬性过滤→必须当天金叉"变为"DIF>DEA多头格局得高分"
2. 三看从"强制全过→一票否决"改为"全过得高分，部分过也得部分分"
3. 资金从"净流入>0才通过"变为"净流入得高分，净流出扣分"
4. 总分仍然100分制，但**设置及格线60分**而非每层硬性过滤
5. 最终输出Top N只，而不是"全过或全不过"

### 方案C：快速修复（最小改动，立竿见影）

如果想最快看到效果，只需改3处：

1. **MACD条件放宽**（L1210-1211）：加一行 `or (dif_now > dea_now)` （多头排列也算）
2. **三看改为二看通过**（L1144-1148）：`all_passed` 改为 `passed_count >= 2`
3. **换手率降低**：默认参数 `min_turnover` 从 3 改为 1.5

## 八、总结

| 问题类别 | 严重程度 | 数量 |
|---------|---------|------|
| 🔴 致命（导致结果为0） | 高 | 3个（MACD金叉瞬时效、三看一票否制、多条件叠加过严） |
| 🟡 重要（严重收窄候选池） | 中 | 4个（换手率偏高、涨幅阈值、板块预过滤、资金硬性要求） |
| 🟢 一般（影响质量但不影响有无） | 低 | 2个（bonus_score超上限、PRD描述不足） |

**核心结论**：趋势突破策略的设计思路是正确的，但**多个独立的高门槛条件以AND关系叠加**，形成了"漏斗死亡"。最关键的修复是将 **MACD从"当天金叉"改为"DIF>DEA多头格局"**，以及将 **三看从"强制全过"改为"加权评分"**。

**推荐实施顺序**：
1. 先做方案C（3行代码改动），验证能否出结果
2. 再做方案B（架构级重构），建立更科学的评分体系
3. 最后同步更新PRD文档
