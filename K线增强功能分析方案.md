# K-line 增强 — 功能点分析与价值评估

> 编制日期：2026-04-22
> 当前版本：日K线基础版（MA5/MA20 + 成交量）
> 目标：多周期 + 多指标 + 交易闭环的完整K线分析体系

---

## 一、当前现状

### 1.1 已实现能力

| 能力 | 状态 | 说明 |
|------|------|------|
| 日K线图 | ✅ | ECharts candlestick，最近60日数据 |
| 前复权 | ✅ | `adj_factor` 计算 |
| MA均线 | ✅ | MA5（橙）+ MA20（蓝） |
| 成交量 | ✅ | 底部柱状图，红绿对应涨跌 |
| 缩放/平移 | ✅ | 鼠标滚轮 + 底部滑块 |
| 悬浮提示 | ✅ | 开/收/低/高/量/MA5/MA20 |

### 1.2 当前存在的明显短板

| 短板 | 影响场景 | 严重程度 |
|------|---------|---------|
| 只有日K，无分钟线 | 盘中决策时无法看当日走势 | 🔴 高 |
| 无技术指标（MACD/KDJ/RSI） | 技术面分析停留在均线层面 | 🔴 高 |
| 均线只有MA5/MA20 | 缺少MA10/MA60/MA250，无法判断中长期的趋势 | 🟡 中 |
| K线上不显示买卖标记 | 无法直观回顾自己的交易时机 | 🟡 中 |
| 无周期切换（日/周/月） | 波段分析和长期趋势查看困难 | 🟡 中 |
| 无画线/标注能力 | 支撑/压力位只能凭记忆 | 🟢 低 |

### 1.3 用户使用场景缺失

```
场景1：上午10点看中一只超跌反弹候选股
  → 打开K线图 → 只能看到"昨日收盘前"的日K
  → ❌ 无法判断今天开盘后是否已反弹，错过了最佳入场时机

场景2：持仓已持有15天，想判断是否该卖出
  → 打开K线图 → 只能看MA5/MA20
  → ❌ 没有MACD背离信号、没有KDJ超买提示
  → 决策依据不足

场景3：回顾3个月前的一笔买入操作
  → 打开K线图 → 看不到自己当时在哪一天、什么价格买入的
  → ❌ 无法复盘交易时机的好坏
```

## 三、逐功能详细分析

### 3.1 P0-1：分钟线（5分钟 / 15分钟 / 60分钟 + 分时图）

**当前痛点：**
- 日K数据有 T+1 延迟（今天是4月22日，看到的"最新K线"其实是4月21日收盘）
- 交易时段（9:30-15:00）无法看到当天实时走势
- 选股结果出来后，无法判断"今天开盘是否已经反弹过了"

**功能设计：**

```
┌──────────────────────────────────────────┐
│  [日K]  [周K]  [月K]  [60分]  [15分]  [5分]  [分时]  │ ← 周期切换Tab
├──────────────────────────────────────────┤
│                                          │
│           K线图主区域                     │
│                                          │
├──────────────────────────────────────────┤
│           成交量 / MACD副图               │
└──────────────────────────────────────────┘
```

| 周期 | 数据范围 | 用途 |
|------|---------|------|
| 分时 | 当日9:30-15:00逐分钟 | 盘中实时监控、判断日内买卖点 |
| 5分钟 | 最近5-10个交易日 | 短线波段（1-3天持仓） |
| 15分钟 | 最近15-30个交易日 | 短线波段（3-7天持仓） |
| 60分钟 | 最近2-3个月 | 中线持仓（1-4周） |
| 日K | 最近3-6个月 | 中长线趋势判断 |
| 周K | 最近1-2年 | 长期趋势 + 大级别支撑压力 |
| 月K | 最近3-5年 | 年线级别趋势 + 历史大底/顶 |

**数据源方案：**

| 数据类型 | 推荐接口 | 说明 |
|---------|---------|------|
| 分时数据 | 东方财富实时行情 | 免费，秒级，已有 `get_eastmoney_quotes()` 基础 |
| 5/15/60分钟历史 | Tushare `stk_mins` | 需确认权限级别 |
| 5/15/60分钟历史 | 东方财富分钟线API | 免费替代方案 |
| 周K/月K | Tushare `weekly`/`monthly` | 已有权限 |

**⚠️ 关键决策：**
> 东方财富的分钟线API是否需要额外的爬虫开发？还是Tushare的 `stk_mins` 已经可用？
> 建议先用Tushare `stk_mins`（freq=5min/15min/60min）测试，如果权限不足再转东方财富。

### 3.2 P0-2：技术指标（MACD / KDJ / RSI / BOLL）

**当前痛点：**
- 卖出智能检查、三看确认、选股引擎都在用MACD/KDJ/RSI做判断
- 但K线图上完全不展示这些指标
- 用户无法直观地看到"MACD是否金叉"、"KDJ是否超买"

**功能设计：**

```
K线图上方：指标切换按钮
[MACD] [KDJ] [RSI] [BOLL] [VOL] ← 点击切换副图指标
```

**指标计算方式：** 全部前端基于K线close/high/low序列计算，零后端开发

| 指标 | 计算公式复杂度 | 可视化方式 |
|------|-------------|-----------|
| **MACD** | 中（EMA12/EMA26/DIF/DEA/柱状图） | 主图+副图：DIF/DEA线 + 红绿柱 |
| **KDJ** | 低（RSV→K→D→J） | 副图：K/D/J三线 + 超买(80)/超卖(20)虚线 |
| **RSI** | 低（14日平均涨跌） | 副图：RSI线 + 30/70分界线 |
| **BOLL** | 中（20日MA ± 2倍标准差） | 主图叠加：上轨/中轨/下轨三条线 |

**与现有策略的呼应：**

```
选股引擎 → 用MACD金叉做买入信号
   ↓
K线图 → 显示MACD，用户亲眼看到金叉发生的位置
   ↓
卖出检查 → 用MACD死叉做卖出信号
   ↓
K线图 → 显示MACD，用户确认死叉
```

这形成了一个**"策略逻辑 ↔ 可视化验证"**的闭环。

---

### 3.3 P1-1：交易标记（买入/卖出点标注）

**当前痛点：**
- 用户打开某只股票的K线图，看不到自己的交易记录
- 无法复盘"当时为什么在这个位置买入"
- 持仓详情里的交易列表和K线图是割裂的

**功能设计：**

```
在K线主图上用特殊标记显示交易记录：

   ▲ ¥45.20  ← 绿色向上箭头 = 买入点，标注价格
   ▼ ¥52.80  ← 红色向下箭头 = 卖出点，标注价格

悬浮箭头 → 显示交易详情：日期/数量/理由
```

**实现方式：**
- 后端：K线API增加 `trades` 参数，返回该股票的买入/卖出记录
- 前端：ECharts `markPoint` 或 `scatter` 系列叠加
- 数据：已有SQLite `trades` 表，零新增数据依赖

**衍生价值：**
- 复盘时一眼看出"买在山顶"还是"买在谷底"
- 多笔交易形成"交易轨迹"，直观展示操作习惯

---

### 3.4 P1-2：均线扩展（MA10 / MA60 / MA120 / MA250）

**当前痛点：**
- 只有MA5/MA20，缺少中长期均线
- 无法判断"股价是否在年线之上"（MA250是重要牛熊分界）
- 三看确认里用到了MA10/MA20/MA60，但K线图上不显示

**功能设计：**

```
均线显隐控制（类似同花顺/东方财富）：
☑ MA5   ☑ MA10   ☐ MA20   ☐ MA60   ☐ MA120   ☐ MA250

默认显示：MA5 + MA20（与现有保持一致）
用户可勾选展开其他均线
```

**各均线含义：**

| 均线 | 颜色建议 | 投资含义 |
|------|---------|---------|
| MA5 | 白色 | 超短期，日内/隔日交易参考 |
| MA10 | 黄色 | 短期，1周内趋势 |
| MA20 | 蓝色（现有） | 中期，月线，1个月内趋势 |
| MA60 | 紫色 | 季度线，3个月趋势，牛熊分界 |
| MA120 | 绿色 | 半年线，6个月趋势 |
| MA250 | 红色 | 年线，一年趋势，长期牛熊分界 |

**成本：** 极低 — 后端已有 `closes` 序列，前端 `rolling mean` 计算即可。

---

### 3.5 P1-3：周期切换（日K / 周K / 月K）

**当前痛点：**
- 固定60日日K，看长期趋势需要疯狂向左滚动
- 判断年线级别支撑/压力位时视野不够

**功能设计：**
- 顶部Tab切换：日K / 周K / 月K
- 日K显示120天数据（现有）
- 周K显示2年数据（约100周）
- 月K显示5年数据（约60月）

**数据源：**
- `pro.weekly(ts_code=..., start_date=..., end_date=...)`
- `pro.monthly(ts_code=..., start_date=..., end_date=...)`

---

### 3.6 P2-1：画线工具（支撑/压力线标注）

**功能：** 用户可以在K线图上手动画线标注支撑/压力位，刷新后保留。

**实现方式：** ECharts 自带 `graphic` 组件支持画线，但**状态持久化**需要后端存储。

**建议：** P2优先级，因为：
- 用户使用频率相对较低
- 需要新增数据库表存储画线数据
- 与系统核心闭环（选股→买入→卖出→复盘）关系较弱

## 四、用户价值总结

### 4.1 按用户类型的价值

| 用户类型 | 最需要的K线增强 | 价值描述 |
|---------|---------------|---------|
| **短线交易者**（持股1-5天） | 分时图 + 5分钟K + MACD | 盘中抓买卖点，分钟级决策 |
| **波段交易者**（持股1-4周） | 日K + 60分钟K + KDJ + 交易标记 | 判断波段高低点，复盘交易时机 |
| **中长线投资者**（持股1月+） | 周K/月K + MA60/MA250 + BOLL | 判断大趋势，避免逆势操作 |
| **复盘学习者** | 交易标记 + 多周期对比 | 回顾操作，提升认知 |

### 4.2 与投资流程的整合价值

```
选股结果 → 打开K线图
              ↓
      ┌──────┼──────┐
      ↓      ↓      ↓
   日K判断  分时确认  指标验证
   趋势方向  盘中入场  MACD/KDJ信号
      ↓      ↓      ↓
      └──────┼──────┘
             ↓
        决策买入/观望
             ↓
        持仓后每日查看K线
             ↓
        交易标记回顾成本位置
             ↓
        MACD/KDJ信号触发卖出检查
             ↓
        卖出后复盘完整交易轨迹
```

## 六、实施建议

### 6.1 推荐实施路线

```
Phase 1（2天）—— 技术指标 + 均线扩展
├── 前端：MACD/KDJ/RSI/BOLL 副图指标切换
├── 前端：MA10/MA60/MA120/MA250 均线显隐控制
└── 价值：让K线图从"能看"变成"可用"

Phase 2（1-2天）—— 交易标记 + 周期切换
├── 后端：K线API追加交易记录
├── 前端：K线上标注买入/卖出箭头
├── 后端+前端：日K/周K/月K切换
└── 价值：形成交易闭环 + 长期趋势视野

Phase 3（2-3天）—— 分钟线 + 分时图
├── 调研：Tushare stk_mins 权限 + 东方财富分钟线API
├── 后端：分钟线数据获取 + API
├── 前端：5/15/60分钟K线切换
├── 后端+前端：分时图（面积图）
└── 价值：盘中决策能力，从"盘后复盘"升级到"盘中实战"

Phase 4（可选，1-2天）—— 增强体验
├── 画线工具（支撑/压力线）
├── K线形态自动识别
└── 价值：锦上添花
```

### 6.2 与超跌反弹v4.0的关系

两个功能可以**并行开发**，互不阻塞：
- 超跌反弹v4.0 → 改动 `screener.py`（选股引擎）
- K线增强 → 改动 `routes/position_bp.py` + `static/js/app.js`（数据展示）

建议 **先启动 K线增强 Phase 1**（技术指标+均线），因为：
1. 开发量更小（2天 vs 超跌反弹Phase A的1-2天）
2. 用户感知更直接（K线图每天看）
3. 后续超跌反弹的MA250等逻辑可以直接在K线图上验证

## 七、结论

| 维度 | 评估 |
|------|------|
| **用户价值** | ⭐⭐⭐⭐⭐ 极高 — K线是每天最高频使用的功能 |
| **技术可行性** | ⭐⭐⭐⭐⭐ 极高 — 数据都有，ECharts能力足够 |
| **开发成本** | ⭐⭐⭐☆☆ 中等 — 约5-7天完整实现 |
| **与现有系统兼容性** | ⭐⭐⭐⭐⭐ 极高 — 纯增量改动，不影响现有功能 |

**核心建议：**
1. **立即启动 Phase 1**（技术指标+均线扩展），2天交付，用户感知最明显
2. **Phase 2**（交易标记+周期切换）紧随其后，1-2天
3. **Phase 3**（分钟线+分时图）需要先做数据源调研（Tushare `stk_mins` 权限确认）
4. 全部完成后，K线图从"基础展示"升级为"专业级分析工具"
