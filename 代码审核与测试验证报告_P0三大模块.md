### 3.2 Vite 版本（frontend/src/）

**❌ 严重问题：App.vue 无法运行**

`frontend/src/App.vue` 的 `<script setup>` 中存在以下问题：

1. **大量变量未定义**：模板中使用了 `tradePlans`、`filteredTradePlans`、`tradePlanStatusText`、`switchToTradePlan`、`loadTradePlans`、`openTradePlanModal` 等变量/函数，但 `<script setup>` 中完全没有定义它们。
2. **仅少量占位符**：只有 `priceAlerts`、`alertNotifications`、`showAlertCenter` 等少量变量被定义为 `ref([])`，但对应的函数（如 `loadPriceAlerts`、`submitPriceAlert`）也未定义（除了底部有几个 price alert 相关的空壳函数）。
3. **与 app.js/setup.js 脱节**：`frontend/src/app.js` 和 `setup.js` 中虽然完整实现了这些逻辑，但 App.vue **没有 import 它们**，导致 Vite 构建版本完全不可用。

**结论**：Vite 版本目前只是一个 HTML 模板壳，所有业务逻辑缺失，直接构建会报大量 `tradePlans is not defined` 等运行时错误。

## 六、修复建议

### 立即修复（P0）

1. **Vite 版 App.vue 修复方案（二选一）：**
   - **方案 A**：将 `app.js` / `setup.js` 中的逻辑合并到 `App.vue` 的 `<script setup>` 中（工作量大，但结构清晰）
   - **方案 B**：在 `App.vue` 中 `import { ... } from './setup.js'` 或 `./app.js`，让 App.vue 专注于模板，逻辑外置（需要先将 app.js 改为 ES Module 导出）
   - **建议**：考虑到双前端同步成本，**优先修复 Flask 版**（已可用），Vite 版可以后续统一重构时再处理。

2. **volume_spike 实现：**
   ```python
   elif a["alert_type"] == "volume_spike":
       # 获取今日成交量与近期平均成交量对比
       try:
           df = pro.daily(ts_code=a["ts_code"], end_date=datetime.now().strftime("%Y%m%d"), limit=20)
           if not df.empty and len(df) >= 2:
               today_vol = float(df.iloc[-1]["vol"])
               avg_vol = float(df["vol"].iloc[-6:-1].mean())  # 近5日均量
               vol_ratio = today_vol / avg_vol if avg_vol > 0 else 0
               if a["direction"] == "above" and vol_ratio >= a["threshold"]:
                   is_triggered = True
                   msg = f"📊 {a.get('name', a['ts_code'])} 成交量异动：{vol_ratio:.1f}倍 ≥ {a['threshold']}倍"
       except Exception:
           pass
   ```

### 建议修复（P1）

3. 在 `update_existing_price_alert` 和 `remove_price_alert` 开头添加存在性检查：
   ```python
   alert = get_price_alert(alert_id, user_id)
   if not alert:
       return jsonify({"error": "价格预警不存在"}), 404
   ```

4. 在 `except Exception:` 块中添加 `print(f"[WARN] 检查触发失败: {e}")`，便于调试。
