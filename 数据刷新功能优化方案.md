# 数据刷新功能优化方案

## 一、现状梳理

### 1.1 当前刷新机制

| 刷新类型 | 触发方式 | 刷新内容 | 频率/时机 |
|---------|---------|---------|----------|
| **自动刷新** | `startAutoRefresh()` | 持仓数据 + 大盘指数 | 每 8 秒 |
| **手动刷新** | 点击"🔄 刷新"按钮 | 持仓数据 + 大盘指数 (+交易记录) | 即时 |
| **Tab切换刷新** | 切换 Tab 时 | 各 Tab 专属数据 | 切换时 |
| **Token刷新** | `startTokenRefresh()` | Access Token | 每小时 |

### 1.2 各 Tab 数据加载时机

| Tab | 数据内容 | 当前加载时机 | 问题 |
|-----|---------|-------------|------|
| **持仓** | 持仓列表、盈亏、市值、总资产 | 页面加载 + 自动刷新(8s) | ✅ 正常 |
| **选股** | 选股结果、策略效果 | 手动点击"执行选股"/"查看胜率" | ⚠️ 策略效果无自动刷新 |
| **复盘** | 交易统计、胜率、盈亏比 | 切换 Tab 时 | ✅ 正常 |
| **策略** | 市场环境、仓位建议 | 切换 Tab 时 | ✅ 正常 |
| **自选** | 观察池列表 | 切换 Tab 时 | ✅ 正常 |
| **市场** | 涨停/北向/龙虎榜/板块 | 切换 Tab 时 | ✅ 正常 |

### 1.3 当前代码分析

```javascript
// 自动刷新（每8秒）
function startAutoRefresh(){
    refreshTimer = setInterval(() => {
        if(isLoggedIn.value){
            fetchData();      // 持仓数据
            fetchIndex();     // 大盘指数
        }
    }, 8000);
}

// 手动刷新
async function refreshData(){
    await Promise.all([fetchData(), fetchIndex()]);
    if(activeTab.value === 'tradelog') await fetchTradeLog();
    showToast('数据已刷新');
}
```

**问题**：
1. 手动刷新只处理持仓、大盘、交易记录，其他 Tab 数据不刷新
2. 选股结果、策略效果、复盘数据等需要切换 Tab 才能更新
3. 8秒自动刷新在交易时段合理，但非交易时段浪费资源

## 二、优化方案

### 2.1 刷新数据范围优化

#### 方案 A：智能 Tab 感知刷新（推荐）

手动刷新按钮根据当前 Tab 智能刷新对应数据：

```javascript
async function refreshData(){
    const tab = activeTab.value;
    
    // 基础数据始终刷新
    await Promise.all([fetchData(), fetchIndex()]);
    
    // 根据当前 Tab 刷新特定数据
    switch(tab){
        case 'tradelog':
            await fetchTradeLog();
            break;
        case 'screener':
            // 选股结果：如果有缓存则刷新，否则不主动执行选股
            if(screenResults.value && screenResults.value.length > 0){
                await runScreen(true); // force=true 强制刷新
            }
            // 策略效果：如果已加载则刷新
            if(spData.value){
                await loadStrategyPerformance(true);
            }
            break;
        case 'review':
            await loadReview(reviewPeriod.value);
            break;
        case 'strategy':
            await Promise.all([loadRegime(), loadStrategyPerformance(true)]);
            break;
        case 'watch':
            await fetchWatchList();
            break;
        case 'market':
            await loadMarketData();
            break;
    }
    
    showToast('数据已刷新');
}
```

#### 方案 B：分层刷新策略

| 层级 | 数据类型 | 刷新策略 |
|-----|---------|---------|
| **L1 核心** | 持仓、大盘、资金 | 自动刷新(8s) + 手动刷新 |
| **L2 重要** | 交易记录、观察池 | 手动刷新 + Tab切换刷新 |
| **L3 计算密集型** | 选股结果、策略效果 | 仅手动触发（避免频繁计算） |
| **L4 静态** | 复盘报告、市场环境 | Tab切换时刷新 |

### 2.2 刷新频率优化

#### 当前问题
- 自动刷新固定 8 秒，非交易时段也持续刷新，浪费资源

#### 优化方案

```javascript
// 智能刷新频率
function getRefreshInterval(){
    if(!isTradeTime()){
        // 非交易时段：每5分钟刷新一次
        return 300000;
    }
    // 交易时段根据市场活跃度调整
    const now = new Date();
    const hour = now.getHours();
    const minute = now.getMinutes();
    
    // 开盘前15分钟(9:15-9:30)：高频刷新
    if(hour === 9 && minute < 30) return 5000;
    
    // 早盘(9:30-11:30)：正常刷新
    if((hour === 9 && minute >= 30) || hour === 10 || (hour === 11 && minute <= 30)) return 8000;
    
    // 午盘(13:00-15:00)：正常刷新
    if((hour === 13) || (hour === 14) || (hour === 15 && minute === 0)) return 8000;
    
    // 其他时间：低频刷新
    return 60000;
}

// 动态调整刷新间隔
let refreshTimer = null;
function startAutoRefresh(){
    const interval = getRefreshInterval();
    refreshTimer = setInterval(async () => {
        if(isLoggedIn.value){
            await fetchData();
            await fetchIndex();
            // 重新计算间隔（适应时段变化）
            const newInterval = getRefreshInterval();
            if(newInterval !== interval){
                clearInterval(refreshTimer);
                startAutoRefresh();
            }
        }
    }, interval);
}
```

### 2.3 刷新状态可视化

增加刷新状态提示，让用户知道数据新鲜度：

```javascript
// 数据新鲜度状态
const dataFreshness = ref({
    positions: null,    // 最后更新时间
    index: null,
    screen: null,
    watch: null
});

// 更新时记录时间
async function fetchData(){
    const r = await fetchWithAuth(`${API}/api/positions`);
    const d = await r.json();
    positions.value = d.positions;
    summary.value = d.summary;
    dataFreshness.value.positions = Date.now();
}
```

UI 显示：
- 🟢 1分钟内："刚刚更新"
- 🟡 1-5分钟："X分钟前"
- 🔴 超过5分钟："数据较旧，建议刷新"

## 三、推荐实现

### 3.1 优先级 P0（立即实施）

1. **智能 Tab 感知刷新**
   - 修改 `refreshData()` 函数，根据当前 Tab 刷新对应数据
   - 避免用户需要切换 Tab 才能看到最新数据

2. **非交易时段降频**
   - 非交易时段（15:00-9:15）自动刷新改为 5 分钟一次
   - 减少服务器压力和流量消耗

### 3.2 优先级 P1（近期实施）

3. **刷新状态可视化**
   - 在数据卡片上显示最后更新时间
   - 数据超过5分钟显示警告提示

4. **手动刷新增强**
   - 添加下拉刷新（移动端）
   - Ctrl+R 快捷键刷新

### 3.3 优先级 P2（后续考虑）

5. **WebSocket 实时推送**
   - 大盘指数、持仓盈亏实时推送
   - 减少轮询，提升实时性
