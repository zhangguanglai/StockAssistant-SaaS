## 1. 产品概述

### 1.1 产品定位

面向个人 A股投资者的智能投研工具，提供**持仓管理、智能选股、策略复盘、K线分析、持仓建议**等全流程功能，帮助投资者提升决策效率。

### 1.2 核心价值矩阵

| 维度 | 描述 | 状态 |
|------|------|------|
| **选股引擎** | 三套策略（趋势突破/板块龙头/超跌反弹），6步筛选+100分评分 | ✅ 已完成 |
| **持仓管理** | 买入/卖出/编辑/删除，多用户隔离，SQLite 持久化 | ✅ 已完成 |
| **策略中心** | 市场环境判断、仓位建议、选股回测、实时预警 | ✅ 已完成 |
| **复盘体系** | 周/月/全周期胜率和盈亏比统计，策略效果跟踪闭环 | ✅ 已完成 |
| **技术分析** | K线图（日K）+ MA均线 + 成交量，支撑/压力位参考 | ✅ 已完成 |
| **市场数据** | 涨停/北向/龙虎榜/板块资金/板块轮动 | ✅ 已完成 |
| **卖出智能检查** | 五维检查(盈亏/技术/价位/影响/风险)，价格参考位，快捷操作 | ✅ v3.4 已完成 |
| **智能预警** | 13个场景自动检测，5级预警(critical→info) | ✅ v3.4.1 已完成 |

### 3.1.1 三看确认系统 ✅

#### 功能概述
在买入股票前，系统自动对目标股票进行"三看"技术面检查，帮助用户判断当前是否适合入场。

| 检查项 | 判定标准 | 数据源 |
|--------|---------|--------|
| **一看：高低点抬高** | 最近3个低点递增 + 最近3个高点递增（n=2窗口极值法） | Tushare日线（前复权）+ 实时行情 → `analyze_hl_points()` |
| **二看：均线多头排列** | MA5 > MA10 > MA20 | Tushare日线（前复权）+ 实时行情 |
| **三看：量价配合** | 量比 > 0.8 且涨幅 < 涨停价 | Tushare日线（前复权）+ 实时行情 |

#### 高低点分析公共函数（v3.5 统一）

全项目所有高低点逻辑统一收敛到 `helpers.py` 的 `analyze_hl_points(highs, lows, n)` 函数：

- **n参数**：极值识别窗口大小（前后各n天），控制灵敏度
- **结构类型**：7种（uptrend/downtrend/sideways/weak_uptrend/weak_downtrend/bottoming/topping）
- **趋势阈值**：2%容忍度（比严格递增更符合实际）
- **输出**：structure/score/signal/high_trend/low_trend/recent_highs/recent_lows/HH-HL-LH-LL统计

#### 数据源策略（v3.4 更新）

```
三看确认数据获取流程：
  ├─ 获取Tushare前复权日线（90天）
  │    └─ adj_factor 前复权因子处理
  ├─ 计算均线/高低点/量比等指标
  ├─ 获取实时行情（新浪/Tushare降级）
  │    └─ 用实时价格替换最后一条日线的close/high/low/open
  ├─ 基于最新数据执行三看检查
  └─ 返回：每项通过状态 + 详细描述 + 最新价 + 数据来源标识
```

#### v3.5 数据源详细规范（2026-04-15）

三看确认弹窗采用 **混合数据源模式**，各指标数据源不同，用户需了解其含义：

| 指标 | 数据源 | 类型 | 实时性 | 说明 |
|------|--------|------|--------|------|
| 股票名称 | Tushare stock_basic | 📅 静态 | 不变 | 基础信息 |
| 最新价格 | ★ 新浪实时接口 (降级Tushare) | ⏱️ **实时** | 盘中变化 | 截图显示"新浪实时"即走此路径 |
| 数据源标签 | `_source` 字段自动检测 | — | 动态 | "新浪实时" / "tushare" |
| **一看：高低点** | Tushare daily high/low 序列 | 📅 **收盘** | T+0.5~1日延迟 | 窗口极值法识别局部极值点 |
| 一看：结构判断 | `analyze_hl_points(n=2)` 计算 | 📅 派生 | 基于收盘序列 | 公共函数统一输出7种结构类型 |
| n值标注 | `_hl_n=2`, `_hl_window_days=5` | 参数 | 固定 | 短线灵敏度窗口 |
| **二看：MA5/10/20** | Tushare daily close + 新浪补丁 | ⚠️ **混合** | 前59根收盘+最后1根实时 | rolling均值计算，最后1根被实时价替换 |
| 二看：多头判断 | MA5 > MA10 > MA20 数值比较 | ⚠️ 派生 | 基于混合均线 | |
| **三看：今日成交量** | Tushare daily vol 字段 | 📅 **昨收** | T+1延迟 | ⚠️ 日线vol通常为前一交易日成交量，非盘中实时量 |
| 三看：5日均量 | vol[-6:-1] 均值 | 📅 历史 | 前5个交易日均值 | |
| 三看：量比 | 今日vol / 5日均量 | 📅 派生 | | |
| 三看：上涨/下跌均量 | 涨跌日分组统计vol均值 | 📅 历史 | 近N日统计 | |
| 总体结论 | 三项AND逻辑综合判断 | 综合 | | |

> **关键注意事项**：
> 1. **高低点和成交量的基础数据来自Tushare日线**，有T+0.5~T+1延迟，盘中可能不是当日最新值
> 2. **只有最新价格(closes[-1])和均线最后1根接入了新浪实时行情**
> 3. **成交量如需盘中实时数据**，需要额外接入付费接口(`stk_realtime` 2000元/年)或东方财富实时接口
> 4. 当前设计已满足"盘前决策"和"非交易时段使用"场景；交易日盘中使用时，高低点和成交量可能有约1天滞后

#### v3.4 修复记录（2026-04-13）
- **问题**：三看确认仅使用Tushare延迟数据，最新价为历史收盘价而非实时价
- **影响**：高低点/均线/量价的判断基于过期价格，结果不可信
- **修复**：
  1. 添加 `get_realtime_quotes()` 调用，用实时价格替换日线最后一条
  2. 添加前复权处理（adj_factor），长期持股分析更准确
  3. 新增返回字段 `realtime_price`、`data_source`
- **文件**：`routes/position_bp.py` - `check_three_views()` 函数

### 3.1.2 卖出智能检查系统 ✅ (v3.5)

#### 功能概述
在用户点击"卖出"时，弹窗自动加载**五维卖出决策辅助数据**，将纯表单升级为智能决策面板。

> 核心理念：**卖出不是填表单，是做决策。**

#### 五维检查体系

| 维度 | 内容 | 作用 |
|------|------|------|
| 🔴 **盈亏状态** | 7级分级（deep_loss→big_profit） | 告诉用户当前该不该卖 |
| 📊 **技术面信号** | 均线排列 + 量价配合 + 连续走势 + 位置分析（4项） | 技术面是否支持卖出 |
| 🎯 **价格参考位** | 8个可点击价位（R1-R5止盈 + 现价 + S1-S3止损） | 提供定价锚点 |
| 💰 **财务影响预估** | 实时联动计算：收入/成本/盈亏/剩余状态 | 卖出后资产变化预览 |
| ⚠️ **风险警示** | 分级警告（critical/danger/warning/caution/info） | 异常信号提醒 |

#### v3.5 数据源详细规范（2026-04-15）

卖出智能检查采用 **混合数据源模式**，各指标数据源如下：

| # | 指标 | 数据源 | 类型 | 实时性 | 说明 |
|---|------|--------|------|--------|------|
| **顶部信息栏** | | | | | |
| 1 | 持仓数量 | SQLite positions表 | 💾 静态 | 不变 | 用户录入的买入数据 |
| 2 | 成本价 | SQLite trades表(加权平均) | 💾 静态 | 不变 | `总买入金额/总买入股数` |
| 3 | 现价 | ★ 新浪实时接口 (降级Tushare) | ⏱️ **实时** | 盘中变化 | `get_realtime_quotes()` 获取 |
| 4 | 盈亏金额 | `(现价-成本)×持仓` | ⏱️ **实时派生** | 随现价变化 | |
| 5 | 盈亏比例 | `(现价-成本)/成本×100%` | ⏱️ **实时派生** | 随现价变化 | |
| **维度一：盈亏状态检查** | | | | | |
| 6 | 7级分级 | 基于profit_pct阈值判断 | ⏱️ 派生 | 实时 | 阈值: >20大赚/>10盈利/>3微利/>-3持平/>-8轻亏/>-15中亏/≤-15深套 |
| **维度二：技术面信号** | | | | | |
| 7a | MA5 | Tushare daily close(前4根) + 新浪补丁(最后1根) | ⚠️ **混合** | 前59根📅+最后1根⏱️ | `sum(closes[-5:])/5` |
| 7b | MA20 | Tushare daily close(前19根) + 新浪补丁(最后1根) | ⚠️ **混合** | 前59根📅+最后1根⏱️ | `sum(closes[-20:])/20` |
| 7c | 均线排列判定 | MA5 vs MA20 数值比较 | ⚠️ 派生 | 基于混合均线 | 6种细分: bullish强势/weak_bearish转弱/bearish破位/空头弱势/空头反弹/bullish反转 |
| 8a | 今日成交量 | Tushare daily vol 字段 `volumes[-1]` | 📅 **昨收** | ⚠️ T+1延迟 | 日线vol通常为前一交易日成交量 |
| 8b | 5日均量 | `sum(volumes[-5:])/5` | 📅 历史 | 前5个交易日均值 | |
| 8c | 量比 | 今日vol / 5日均量 | 📅 派生 | | |
| 8d | 日涨跌幅(pct_1d) | `(closes[-1]-closes[-2])/closes[-2]*100` | ⚠️ **混合派生** | closes[-1]若被实时替换则含实时成分 | v3.4修复：不再用实时pct_chg(不复权) |
| 8e | 量价配合判定 | 量比+日涨跌幅组合规则 | 📅 派生 | | 5级: volume_down危险/volume_up强势/weak_down缩量/low_vol清淡/normal正常 |
| 9a | 连续涨跌 | closes序列方向统计(阈值0.01元) | ⚠️ **混合** | 最后1根可能为实时价 | 从最近一天向前找首个非平盘方向后的连续同向天数 |
| 9b | 连续走势判定 | consec_up/consec_down 计数 | ⚠️ 派生 | | 5级: overbought过热(≥5天涨)/strong_up连涨/weak_down连跌/oversold超跌(≥5天跌)/normal正常 |

| 10b | 30日最低价 | `min(lows[-30:])` | 📅 **收盘** | T+1延迟 | Tushare daily low |
| 10c | 距高点距离 | `(high_30d-latest)/latest*100` | ⚠️ **混合派生** | latest为实时价 | |
| 10d | 距低点距离 | `(latest-low_30d)/latest*100` | ⚠️ **混合派生** | latest为实时价 | |
| 10e | 位置判定 | dist_to_high/dist_to_low 组合规则 | ⚠️ 派生 | | 5级: near_bottom(<3%)/low_area(3-8%)/neutral中间/high_area(3-8%)/near_top(<3%) |
| **维度三：价格参考位** | | | | | |
| 11 | S1 近30日最低 | `min(lows[-30:])` | 📅 收盘 | | 强支撑位 |
| 12 | S2 MA20均线 | MA20值 | ⚠️ 混合 | | 减仓观察位 |
| 13 | S3 成本-5% | `avg_cost * 0.95` | 💾 静态 | | 止损底线 |
| 14 | R1 近30日最高 | `max(highs[-30:])` | 📅 收盘 | | 激进止盈位 |
| 15 | R2 MA20+10% | `MA20 * 1.10` | ⚠️ 混合 | | 合理止盈区 |

| 17 | R4 成本+10% | `avg_cost * 1.10` | 💾 静态 | | 标准止盈目标 |
| 18 | R5 成本+20% | `avg_cost * 1.20` | 💾 静态 | | 理想收益目标 |
| **维度四：风险警报** | | | | | |
| 19 | 风险信号列表 | 综合ma_sell/vp_sell/pl_code/cs_sell | 综合 | 动态 | 多重危险叠加触发critical级别 |

> **关键发现**：
> 1. **仅现价、盈亏、R3为纯实时数据**；均线/位置/日涨跌幅为**混合型**（取决于最新K线是否被实时替换）
> 2. **成交量(volumes)全部来自Tushare日线**，有T+1延迟，盘中不是当日真实成交
> 3. **高低点(highs/lows)** 全部来自Tushare日线收盘数据，非盘中极值
> 4. 与三看确认的数据源策略完全一致（同一函数 `get_realtime_quotes()`）

#### 资深投资专家审查意见（v3.5）

##### ✅ 设计合理的部分

| 项目 | 评价 | 说明 |
|------|------|------|
| **盈亏7级分级** | ✅ 合理 | 阈值(-15/-8/-3/+3/+10/+20)符合A股日均波动(~2%)和波段振幅(~10-20%)实际 |

| **基础分60+40浮动** | ✅ 合理 | 避免"全通过=100分"的过度乐观偏差，中性偏持有更安全 |

#### 技术面信号详细算法

##### 1. 均线排列（MA_status）
| 信号 | 条件 | 判断 | 评分影响 |
|------|------|------|---------|
| bullish（多头排列） | MA5 > MA20 | ✅ 通过，不触发卖出 | +15分 |
| neutral（中性） | MA5 ≈ MA20（±2%） | ✅ 通过 | +15分 |
| bearish（空头排列） | MA5 < MA20 | ❌ 不通过，建议卖出 | -10分 |

##### 2. 量价配合（Volume_Price）
| 信号 | 条件 | 判断 | 评分影响 |
|------|------|------|---------|
| volume_up（量价齐升） | 量比 > 1.3 且 日涨跌 > 0 | ✅ 通过 | +15分 |
| normal（正常） | 量比 0.8-1.2 | ✅ 通过 | +15分 |
| volume_down（放量下跌） | 量比 > 1.3 且 日涨跌 < 0 | ❌ 不通过，危险信号 | -15分 |

##### 3. 连续走势（Consecutive）
基于最近10个交易日收盘价，从最新日期向前统计连续涨跌天数：

```
遍历最近10日收盘价（从最新往前）:
  当日 > 前日 → 连涨天数+1，连跌天数归零
  当日 < 前日 → 连跌天数+1，连涨天数归零
  当日 = 前日 → 终止统计
```

| 信号 | 条件 | 判断 | 评分影响 |
|------|------|------|---------|
| overbought（连涨过热） | 连涨 ≥ 5天 | ❌ 不通过，建议卖出 | -10分 |
| strong_up（连涨强势） | 连涨 3-4天 | ✅ 通过 | +10分 |
| weak_down（连跌弱势） | 连跌 3-4天 | ❌ 不通过，建议卖出 | -10分 |
| oversold（连跌超跌） | 连跌 ≥ 5天 | ✅ 通过（超跌反弹预期） | +10分 |
| normal（正常） | 其他情况 | ✅ 通过 | +10分 |

##### 4. 位置分析（Position_Analysis）
基于最近30日K线高低点，计算当前价格相对位置：

```
high_30d = 最近30日最高价
low_30d = 最近30日最低价
dist_to_high = (high_30d - 最新价) / 最新价 × 100%  // 距高点百分比
dist_to_low = (最新价 - low_30d) / 最新价 × 100%    // 距低点百分比
```

| 信号 | 条件 | 判断 | 评分影响 |
|------|------|------|---------|
| near_bottom（贴近底部） | 距低点 < 3% | ✅ 通过，反弹概率高 | +10分 |
| low_area（偏低位置） | 距低点 3%-8% | ✅ 通过 | +10分 |
| neutral（中间位置） | 其他情况 | ✅ 通过 | +10分 |
| high_area（偏高位置） | 距高点 3%-8% | ❌ 不通过，建议卖出 | -10分 |
| near_top（贴近顶部） | 距高点 < 3% | ❌ 不通过，建议卖出 | -10分 |

#### 综合评分计算公式 (v3.4.3 保守化调整 - 方案1)

```
基础分 = 60分（满分100可达，保留用户习惯）

均线排列:   不触发卖出 +10分  /  触发卖出 -20分  (失败惩罚翻倍)
量价配合:   不触发卖出 +10分  /  触发卖出 -25分  (放量下跌极危险，扣分最重)
连续走势:   不触发卖出 +10分  /  触发卖出 -15分  (失败惩罚 > 通过奖励)
位置分析:   不触发卖出 +10分  /  触发卖出 -10分  (对称)

总分 = 基础分 + 各维度得分
最终分 = clamp(总分, 0, 100)  // 限制在0-100之间
```

**评分区间与操作建议（5档细分）：**
| 分数区间 | 建议 | 标签 | 颜色 |
|---------|------|------|------|
| 90-100 | 强烈持有 | 技术面优秀，安心持有 | 🟢 |
| 75-89 | 正常持有 | 技术面正常，可继续持有 | 🟢 |
| 50-74 | 观望/减仓 | 出现部分风险信号，建议观望或减仓 | 🟡 |
| 25-49 | 建议减仓 | 风险信号明显，建议减仓 | 🟠 |
| 0-24 | 建议清仓 | 强烈卖出信号，建议清仓 | 🔴 |

**评分范围分析：**
- 最高分 = 60 + 10 + 10 + 10 + 10 = **100分**（全部通过）✓
- 最低分 = 60 - 20 - 25 - 15 - 10 = **-10分 → 钳制为0分**（全部触发卖出）
- 调整目的：保留满分100的用户习惯，通过加重失败惩罚实现保守化，避免过度乐观

##### 各维度详细评分细则

**一、趋势面（35分）— trend_scoring()**
- **趋势强度(13分)**=MA20位置(5)+方向(4)+HL(4)：站上且偏离<=max(1,参数)=5最健康；偏离<=max(3,参数*3)=4；<=6%=2；>6%=1；未站上=0。方向：连升3日=4/升2日=3/升1日=2/走平或降=0。HL：双抬=4/单抬=2-3/震荡=1
- **MACD状态(12分)**：零上金叉(DIF穿DEA且柱转正且零上)=12 / 多头格局(DIF>DEA且DEA>0)=11 / 弱多头(DIF>DEA)=7 / 接近金叉(DIF>DEA_prev)=3 / 空头=0
- **均线多头排列(10分)**：MA5>MA10>MA20 且发散>2%=完美10 / 发散>0.5%=标准8 / 粘合=6 / 部分(MA5>MA20)=4 / 混乱=0
- *辅助联动加分(不占35分额度)*：量比达标→+0.5~1；涨幅达标→+0~1.5

**二、板块面（15分 v3.1）— sector_money_scoring()**
- **v3.1变化**：原满分20分（相对强弱12+排名8）→ 压缩到15分：`sector_score = int(score × 15 / 20)`
- **相对强弱(12分)**=板块涨跌幅 - 上证指数涨跌幅：超额>3%=12 / >1.5%=10 / >0.5%=7 / >-0.5%(持平)=4 / >-2%=2 / 更弱=0
- **排名(8分)**：概念榜前3=8 / 前6=6 / 前10=4 / 前15=2
- **数据源**：Tushare `ths_daily` 或东方财富 → **T+1延迟**，仅反映上一交易日板块表现

**三、资金面（10分 v3.1）— sector_money_scoring() 续**
- **v3.1变化**：原满分20分 → 压缩到10分：`money_score = int(min(score, 20) × 10 / 20)`
- **流入强度(8分)**：流入比率(近5日净流入/流通市值)>2%=8 / >1%=6 / >0.3%=4 / 微正=2；净流出>-1%也保底2分
- **大单净买结构(7分)**⭐：(大买-大卖)/总大单占比 >10%=7 / >5%=5 / >0%=3 / 均衡(-5%~0)=1 / 偏空=0
- **持续性(5分)**：>=参数门槛时4天+=5 / 3天=4 / 2天=3 / 1天=2；不达标也给1分保底
- **数据源**：Tushare `moneyflow` → **T+1延迟**，仅反映上一交易日资金流

**四、多周期共振（15分）— resonance_scoring()**
- 三看通过数 → 得分：3/3全过=🔥15 / 2/3过半=9 / 1/3部分=5 / 0/3不过=0
- 三看定义：一看高低点抬高(HH/LH)、二看均线多头(MA5>MA10>MA20)、三看量价配合(量比>0.8且上涨日量>下跌日量)

**五、催化加分（10分 capped）— catalyst_scoring()**
- **基本面(5分)**：业绩预告预增/扭亏/略增(30日内)=5；回购>1亿=5；互斥取高；回购>3000万=3

- **情绪(2分)**：近5日曾出现在涨停板名单(limit_list接口)
