## 1. 产品概述

### 1.1 产品定位

面向个人 A股投资者的智能投研工具，提供**持仓管理、智能选股、策略复盘、K线分析、持仓建议**等全流程功能，帮助投资者提升决策效率。

### 1.2 核心价值矩阵

| 维度 | 描述 | 状态 |
|------|------|------|
| **选股引擎** | 三套策略（趋势突破/板块龙头/超跌反弹），6步筛选+100分评分 | ✅ 已完成 |
| **持仓管理** | 买入/卖出/编辑/删除，多用户隔离，SQLite 持久化 | ✅ 已完成 |
| **策略中心** | 市场环境判断、仓位建议、选股回测、实时预警 | ✅ 已完成 |
| **复盘体系** | 周/月/全周期胜率和盈亏比统计，策略效果跟踪闭环 | ✅ 已完成 |
| **技术分析** | K线图（日K）+ MA均线 + 成交量，支撑/压力位参考 | ✅ 已完成 |
| **市场数据** | 涨停/北向/龙虎榜/板块资金/板块轮动 | ✅ 已完成 |
| **卖出智能检查** | 五维检查(盈亏/技术/价位/影响/风险)，价格参考位，快捷操作 | ✅ v3.4 已完成 |
| **智能预警** | 13个场景自动检测，5级预警(critical→info) | ✅ v3.4.1 已完成 |

### 3.1.1 三看确认系统 ✅

#### 功能概述
在买入股票前，系统自动对目标股票进行"三看"技术面检查，帮助用户判断当前是否适合入场。

| 检查项 | 判定标准 | 数据源 |
|--------|---------|--------|
| **一看：高低点抬高** | 最近3个低点递增 + 最近3个高点递增 | Tushare日线（前复权）+ 实时行情 |
| **二看：均线多头排列** | MA5 > MA10 > MA20 | Tushare日线（前复权）+ 实时行情 |
| **三看：量价配合** | 量比 > 0.8 且涨幅 < 涨停价 | Tushare日线（前复权）+ 实时行情 |

#### 数据源策略（v3.4 更新）

```
三看确认数据获取流程：
  ├─ 获取Tushare前复权日线（90天）
  │    └─ adj_factor 前复权因子处理
  ├─ 计算均线/高低点/量比等指标
  ├─ 获取实时行情（新浪/Tushare降级）
  │    └─ 用实时价格替换最后一条日线的close/high/low/open
  ├─ 基于最新数据执行三看检查
  └─ 返回：每项通过状态 + 详细描述 + 最新价 + 数据来源标识
```

#### API 返回字段

```json
{
  "ts_code": "603588.SH",
  "name": "高能环境",
  "trade_date": "2026-04-10",
  "close": 16.61,
  "realtime_price": 16.78,
  "data_source": "sina",
  "checks": {
    "high_low": { "passed": true, "description": "高低点抬高", ... },
    "ma": { "passed": true, "description": "均线多头排列", ... },
    "volume": { "passed": false, "description": "量价配合不佳", ... }
  }
}
```

#### v3.4 修复记录（2026-04-13）
- **问题**：三看确认仅使用Tushare延迟数据，最新价为历史收盘价而非实时价
- **影响**：高低点/均线/量价的判断基于过期价格，结果不可信
- **修复**：
  1. 添加 `get_realtime_quotes()` 调用，用实时价格替换日线最后一条
  2. 添加前复权处理（adj_factor），长期持股分析更准确
  3. 新增返回字段 `realtime_price`、`data_source`
- **文件**：`routes/position_bp.py` - `check_three_views()` 函数

### 3.1.2 卖出智能检查系统 ✅ (v3.4)

#### 功能概述
在用户点击"卖出"时，弹窗自动加载**五维卖出决策辅助数据**，将纯表单升级为智能决策面板。

> 核心理念：**卖出不是填表单，是做决策。**

#### 五维检查体系

| 维度 | 内容 | 作用 |
|------|------|------|
| 🔴 **盈亏状态** | 7级分级（deep_loss→big_profit） | 告诉用户当前该不该卖 |
| 📊 **技术面信号** | 均线排列 + 量价配合 + 连续走势 + 位置分析（4项） | 技术面是否支持卖出 |
| 🎯 **价格参考位** | 8个可点击价位（R1-R5止盈 + 现价 + S1-S3止损） | 提供定价锚点 |
| 💰 **财务影响预估** | 实时联动计算：收入/成本/盈亏/剩余状态 | 卖出后资产变化预览 |
| ⚠️ **风险警示** | 分级警告（critical/danger/warning/caution/info） | 异常信号提醒 |

#### 价格参考位设计

```
压力位(止盈)          支撑位(止损)
─────────            ─────────
R1 近30日最高         S2 MA20均线     ← 减仓观察位
R2 MA20+10%          S1 近30日最低    ← 止损底线位
R3 成本+10%          S3 成本-5%
R4 成本+20%          当前价           ← 即时成交位
R5 涨停极限(+10%)    
```
点击任意价格按钮 → 自动填入卖出价格框。

#### 快捷操作

| 操作 | 功能 |
|------|------|
| 全部清仓 | 数量=全部持仓 |
| 减半卖出 | 数量=持仓/2 |
| 止损离场 | 使用S1/S2支撑位 |

#### API 接口

```
GET /api/positions/<id>/sell-check
```

**返回结构**：
```json
{
  "position": { "id", "ts_code", "name", "avg_cost", "current_price", "profit_pct", ... },
  "profit_check": { "level", "label", "pct", "amount", "color", "suggestion", "detail" },
  "technical_check": {
    "ma_status": { "signal"(bullish/neutral/bearish), "label", "passed", "suggest_sell" },
    "volume_price": { "signal", "vol_ratio", "passed", "suggest_sell" },
    "consecutive": { "days_up/down", "signal", "passed", "suggest_sell" },
    "position_analysis": { "dist_to_high/low_pct", "signal", "passed", "suggest_sell" },
    "overall": { "score"(0-100), "verdict"(hold/reduce/sell/sell_strong) }
  },
  "price_levels": {
    "resistances": [ { "label", "price", "type" } ],
    "supports": [ { "label", "price", "type" } ]
  },
  "risk_alerts": [ { "level", "icon", "text" } ],   // 可能为空
  "quick_actions": [ { "action", "label", "volume" } ]
}
```

#### UI 交互流程
```
点击"卖" → 弹窗打开(显示基础信息)
         → 并行加载 sell-check API (≤3秒)
         → 渲染完整智能面板:
              ├ 盈亏状态卡片 (颜色+建议文案)
              ├ 技术面4项信号 (✅通过 / ❌不通过)
              ├ 8个价格参考位 (点击填入)
              ├ 快捷操作按钮 (全部/减半/止损)
              ├ 财务影响预估值 (实时联动)
              └ 风险警报列表 (按级别排序)
         → 用户填写数量/理由
         → 勾选确认框 → 提交
```

### 3.1.3 智能预警系统 ✅ (v3.4.1)

#### 功能概述
在持仓列表中自动展示每只股票的**预警信息**，基于多维度数据实时计算。每只最多显示3条预警，按优先级排序。

#### 预警场景全景（13个场景，5个级别）

##### 维度一：盈亏状态（4个）

| # | 场景 | 级别 | 触发条件 | 显示示例 |
|---|------|------|---------|---------|
| 1 | 深度套牢 | 🔴 danger | 浮亏 ≤ -15% | `深度套牢 -16.2%（¥-3,240）` |
| 2 | 中度亏损 | 🟡 warning | 浮亏 ≤ -8% | `中度亏损 -8.5%，接近止损线` |
| 3 | 丰厚盈利 | ℹ️ info | 浮盈 ≥ +20% | `丰盛盈利 +22.1% 🎉，考虑分批止盈` |
| 4 | 稳健盈利 | ℹ️ info | 浮盈 ≥ +10% | `稳健盈利 +12.3%，关注趋势变化` |

##### 维度二：持仓时长（2个）

| # | 场景 | 级别 | 触发条件 | 显示示例 |
|---|------|------|---------|---------|
| 5 | 长期持仓 | ℹ️ info | 持有 >90天 | `已持有120天（4个月+）` |
| 6 | 僵持亏损 | 🟡 warning | >30天且仍亏>5% | `持45天仍亏-6%，需审视逻辑` |

##### 维度三：技术面异动（2个）

| # | 场景 | 级别 | 触发条件 | 显示示例 |
|---|------|------|---------|---------|
| 7 | 放量破位(强) | 🔴 danger | 价格<MA20 且 量比>1.5 | `⚡ 放量破位！跌穿MA20(-2.3%↓) 量比1.8` |
| 8 | 跌破MA60 | 🟡 warning | 价格 < 长期均线MA60 | `跌破长期均线MA60 (-3.1%↓)` |

##### 维度四：用户设定触发（3个）

| # | 场景 | 级别 | 触发条件 | 显示示例 |
|---|------|------|---------|---------|
| 9 | 止损触发 | 🔴🚨 critical | 当前价 ≤ 用户设定的止损价 | `🚨 已跌破止损价 ¥45.00` |
| 10 | 止盈触达 | ℹ️ info | 当前价 ≥ 止盈价 | `已触达止盈价 ¥55.00 🎯` |
| 11 | 接近止损 | 🟠 caution | 距止损价≤10% | `距止损价仅 3.2%` |

##### 维度五：市场环境（2个）

| # | 场景 | 级别 | 触发条件 | 显示示例 |
|---|------|------|---------|---------|
| 12 | 单日大跌 | 🟡 warning | 今日跌 ≥ -5% | `今日大跌 -5.2% ⚠️` |
| 13 | 板块异动 | 🟠/ℹ️ | 所属板块涨跌幅≥±3% | `💧 板块异动：银行暴跌-3.2%` |

##### 运行规则

| 规则 | 说明 |
|------|------|
| 最多显示3条 | 按 critical > danger > warning > caution > info 排序 |
| 数据源 | 实时行情（新浪/Tushare），交易时段30s刷新 |
| 展示位置 | 持仓列表"预警"列，鼠标悬停显示 detail |
| 颜色编码 | 🔴🚨critical(深红脉冲) / 🔴danger(红) / 🟡warning(黄) / 🟠caution(橙) / ℹ️info(蓝) |
