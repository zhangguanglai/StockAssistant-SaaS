## 1. 产品概述

### 1.1 产品定位

面向个人 A股投资者的智能投研工具，提供持仓管理、智能选股、策略复盘、K线分析、持仓建议等全流程功能，帮助投资者提升决策效率。

### 1.2 核心价值

| 维度 | 描述 |
|------|------|
| **选股引擎** | 三套策略（趋势突破/板块龙头/超跌反弹），6步筛选+100分评分 |
| **持仓管理** | 买入/卖出/编辑/删除，多用户隔离，SQLite 持久化 |
| **策略中心** | 市场环境判断、仓位建议、选股回测、实时预警 |
| **复盘体系** | 周/月/全周期胜率和盈亏比统计，策略效果跟踪闭环 |
| **技术分析** | K线图（日K）+ MA均线 + 成交量，支撑/压力位参考 |

## 2. 核心功能模块

### 2.1 持仓管理

- 持仓 CRUD（创建/编辑/删除/清仓）
- 买入/卖出交易记录
- 实时行情自动填充现价
- 资金管理（总资产/可用资金/持仓市值/浮动盈亏）
- 资产分布饼图（含现金扇区）

### 2.2 选股引擎

| 策略 | 适用市场环境 | 核心逻辑 |
|------|-------------|---------|
| **趋势突破** `trend_break` | 上升/震荡期 | MA20站稳 + MACD金叉 + 放量 + 板块效应 |
| **板块龙头** `sector_leader` | 情绪亢奋期 | 板块涨>2% + 涨幅5-9.5% + 换手5-15% + 主力流入 |
| **超跌反弹** `oversold_bounce` | 下跌末期 | 近20日跌>20% + 止跌信号 + 市值>100亿 |

- **统一4维评分**：趋势/板块/资金/加分 = total_score（0-100）
- **策略参数可调**：每个策略5-7个核心参数，运行前弹窗确认/修改
- **选股历史去重**：按 (date, time, strategy) 三元组去重

#### 2.2.1 板块龙头首板策略（sector_leader）详细设计

**策略定位**

捕捉板块爆发初期的龙头股，在板块启动确认后、个股尚未涨停前介入，博取次日溢价或连板收益。

**核心规则**

| 规则维度 | 条件 | 参数名 | 默认值 | 可调范围 |
|---------|------|--------|--------|---------|
| **板块热度** | 概念板块涨幅 ≥ 阈值 | `min_board_pct` | 2% | 1-5% |
| **个股涨幅** | 涨幅在区间内（未涨停） | `min_stock_pct` / `max_stock_pct` | 5% / 9.5% | 3-7% / 7-9.9% |
| **换手充分** | 换手率在区间内 | `min_turnover` / `max_turnover` | 5% / 15% | 3-8% / 10-25% |
| **市值适中** | 流通市值在区间内 | `min_circ_mv` / `max_circ_mv` | 30亿 / 200亿 | 10-50亿 / 100-500亿 |
| **非涨停股** | 排除已涨停股票 | - | - | - |
| **资金流入** | 主力资金净流入 > 0 | - | - | - |

**算法逻辑**

```
1. 获取概念板块涨幅排行（Tushare ths_daily）
   └─ 筛选涨幅 ≥ min_board_pct 的热门板块，取Top15

2. 获取热门板块成分股（东方财富 clist 接口）
   └─ 遍历每个热门板块，获取板块内所有成分股
   └─ 筛选涨幅在 [min_stock_pct, max_stock_pct] 区间的股票
   └─ 构建候选池 candidate_pool

3. 【降级模式】如果东方财富接口不可用
   └─ 使用 Tushare daily_basic 批量筛选全市场
   └─ 条件：涨幅区间 + 换手率区间 + 市值区间

4. 多维度评分（100分制）

   | 维度 | 权重 | 计算逻辑 |
   |------|------|---------|
   | 涨换分 | 0-40 | 涨幅8-9.5% +20分，6-8% +15分，5-6% +8分；换手8-12% +20分 |
   | 板块分 | 0-25 | 所属板块涨幅Top1 +25分，Top3 +20分，Top5 +15分 |
   | 资金分 | 0-20 | 主力净流入/流通市值比率排名，Top10% +20分 |
   | 加分项 | 0-15 | 近5日有涨停 +10分，近3日资金持续流入 +5分 |

5. 过滤与排序
   └─ 排除近5日已涨停的股票（避免追高）
   └─ 按 total_score 降序排列
   └─ 取 TopN 作为最终输出
```

**数据源支持**

| 数据类型 | 主数据源 | 降级数据源1 | 降级数据源2 | 字段映射 |
|---------|---------|------------|------------|---------|
| 概念板块涨幅 | Tushare `ths_daily` | 缓存数据 | - | `pct_change` |
| 板块成分股 | 东方财富 `clist` | Tushare `ths_member` | - | `f12`→ts_code |
| 个股涨幅 | 东方财富 `f3` | **新浪财经** `pct_chg` | Tushare `daily_basic.pct_chg` | `pct_chg` |
| 个股现价 | 东方财富 `f2` | **新浪财经** `price` | Tushare `daily_basic.close` | `price` |
| 换手率 | 东方财富 `f8` | - | Tushare `daily_basic.turnover_rate` | `turnover_rate` |
| 流通市值 | 东方财富 `f20` | - | Tushare `daily_basic.circ_mv` | `circ_mv` |
| 资金流向 | Tushare `moneyflow` | - | - | `main_net_in` |
| 涨停列表 | Tushare `stk_limit` | - | - | `up_limit` |

**新浪财经实时接口说明：**

- **接口地址**：`https://hq.sinajs.cn/list=`
- **调用方式**：批量查询（逗号分隔股票代码）
- **返回格式**：JavaScript 变量赋值格式
- **字段示例**：`股票名称,今日开盘价,昨日收盘价,当前价,今日最高价,今日最低价,...`
- **特点**：
  - 支持批量查询（一次最多约 800 只）
  - 返回数据包含：现价、涨跌幅、成交量、成交额等
  - 当前服务器网络可直接访问，作为东方财富降级方案
  - 需添加 `Referer: https://finance.sina.com.cn` 请求头

**降级机制**

```
实时行情获取
  ├─ 东方财富接口
  │    ├─ 成功 → 返回东方财富数据
  │    └─ 失败 → 降级到新浪财经
  │         ├─ 成功 → 返回新浪财经数据
  │         └─ 失败 → 降级到 Tushare 日线
  └─ 非交易时段 → 直接使用 Tushare 日线

概念板块数据获取
  ├─ 成功 → 继续获取成分股
  │    └─ 东方财富成分股接口
  │         ├─ 成功 → 正常模式筛选
  │         └─ 失败 → 降级模式：用 daily_basic 全市场筛选
  └─ 失败 → 使用缓存的板块数据
       └─ 仍失败 → 完全降级：仅用 daily_basic 按涨幅筛选
```

**输出字段**

```json
{
  "ts_code": "股票代码",
  "name": "股票名称",
  "price": "当前价格",
  "pct_chg": "涨跌幅",
  "turnover_rate": "换手率",
  "circ_mv_yi": "流通市值（亿元）",
  "max_board_name": "所属最强板块",
  "max_board_pct": "板块涨幅",
  "concepts": ["概念1", "概念2"],
  "trend_score": "涨换分",
  "sector_score": "板块分",
  "money_score": "资金分",
  "bonus_score": "加分",
  "total_score": "总分"
}
```

**适用场景与风险提示**

| 场景 | 建议 |
|------|------|
| 涨停家数 > 50 家 | 策略效果最佳，市场情绪亢奋 |
| 板块涨幅 > 5% | 板块爆发日，优先选择板块内涨幅5-9%个股 |
| 震荡市 | 谨慎使用，控制仓位 |
| 下跌市 | 不建议使用，逆势风险高 |

**风险因素**：板块持续性不确定、次日低开风险、追高风险（需严格按涨幅区间筛选）

### 2.3 持仓建议

- 5维评分（±100分）→ 加仓/持有/观望/减仓/清仓
- 止损止盈弹窗：6个支撑/压力位参考，点击自动填入

### 2.4 观察池 & 策略效果跟踪

- 选股结果可加入观察池跟踪后续表现
- 按策略分组统计胜率/均收益/评分区间
- 选股 → 观察池 → 胜率验证的完整闭环

### 2.5 实时预警

- 前端每60秒自动检查持仓变化
- 浏览器 Notification API 桌面通知

##### `pro_bar` 分钟线

| 项目 | 说明 |
|------|------|

| 场景 | 可行性 | 说明 |
|------|--------|------|

#### 3.3.4 数据源选择策略

系统采用**三级数据源智能降级**机制：

```
请求进入 → should_use_realtime_source()
  ├─ YES（工作日 9:15-23:59）→ 东方财富（实时）
  │    ├─ 成功 → 返回东方财富数据
  │    └─ 失败 → 降级到新浪财经（实时）
  │         └─ 失败 → 降级到 Tushare（历史）
  └─ NO（周末/凌晨）→ Tushare（历史）
```

| 时间段 | 优先级1 | 优先级2 | 优先级3 | 原因 |
|--------|---------|---------|---------|------|
| 工作日 9:15-15:00 | 东方财富 | 新浪财经 | Tushare | 交易时段需要实时数据 |
| 工作日 15:00-23:59 | 东方财富 | 新浪财经 | Tushare | 收盘后仍返回当天收盘数据 |
| 凌晨 0:00-9:15 | Tushare | - | - | 无实时交易 |
| 周末全天 | Tushare | - | - | 无实时数据 |

**数据源特性对比：**

| 数据源 | 实时性 | 覆盖范围 | 稳定性 | 适用场景 |
|--------|--------|---------|--------|---------|
| **东方财富** | 实时 | 沪深A股、指数、板块 | 依赖网络环境 | 首选实时源 |
| **新浪财经** | 实时 | 沪深A股、指数（批量支持） | 高（服务器可访问） | 东方财富降级 |
| **Tushare** | 日线（T-1） | 全市场历史数据 | 高（API稳定） | 非交易时段、历史计算 |

**缓存策略（v3.2）：**

| 数据类型 | 交易时段 TTL | 非交易时段 TTL |
|---------|-------------|--------------|
| 东方财富行情 | 30秒 | 5分钟 |
| 大盘指数 | 30秒 | 5分钟 |
| `/api/index` 路由 | 30秒 | 5分钟 |
