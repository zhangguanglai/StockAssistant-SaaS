# 选股模块交互体验优化建议

## 一、现状问题分析

### 1.1 信息架构问题

| 问题 | 描述 | 影响 |
|------|------|------|
| 策略选择器占用空间过大 | 3个策略按钮横向排列，每个160px最小宽度 | 小屏幕下换行混乱 |
| 策略详情与选择器分离 | 选择策略后，详情显示在下方独立区域 | 用户难以建立选择-反馈关联 |
| 筛选漏斗视觉混乱 | 使用箭头符号(→)连接，缺乏视觉层次 | 难以快速理解筛选流程 |
| 结果表格信息过载 | 单行列包含9个字段 | 用户注意力分散 |

### 1.2 交互流程问题

| 问题 | 描述 | 影响 |
|------|------|------|
| 缺少参数确认环节 | 点击选股直接执行，无参数预览 | 用户无法预知筛选条件 |
| 运行状态反馈不足 | 只有文字提示"预计3-5分钟" | 用户焦虑感强，无法预估进度 |
| 结果排序功能缺失 | 无法按评分/涨幅/市值排序 | 用户难以按需查看结果 |
| 批量操作入口不明显 | "Top 10加入观察池"按钮在表格上方 | 容易被忽略 |

### 1.3 视觉设计问题

| 问题 | 描述 | 影响 |
|------|------|------|
| 策略卡片无选中态 | 仅边框颜色变化，不够明显 | 用户不确定当前选中策略 |
| 评分展示不直观 | 评分明细以文字形式展示 | 难以快速比较不同股票 |
| 高评分股票标识弱 | 仅左侧红色边框，无其他强调 | 优质股票不够突出 |
| 空状态设计简陋 | 仅显示文字"暂无入选股票" | 缺乏引导性操作提示 |

## 二、优化建议（按优先级排序）

### P0 - 核心体验优化

#### 1. 策略选择器重构

**当前设计：**
```
[趋势突破] [板块龙头] [超跌反弹]  ← 横向按钮组
策略详情：...
```

**建议设计：**
```
┌─────────────────────────────────────┐
│  🔥 趋势突破    适合：上升/震荡期     │  ← 卡片式，选中态更明显
│  突破MA20+MACD金叉+放量突破...       │
│  📌 持有5天  🛡️ 止损-5%  💰 低吸    │
├─────────────────────────────────────┤
│  ⚡ 板块龙头    适合：情绪亢奋期     │  ← 未选中态为灰色
│  ⚡ 超跌反弹    适合：下跌末期       │
└─────────────────────────────────────┘
```

**实现要点：**
- 采用垂直卡片式布局，选中卡片展开显示详情
- 未选中卡片保持紧凑，仅显示名称+适用场景
- 选中态使用背景色+左侧边框双重强调

#### 2. 增加参数确认弹窗

**触发时机：** 点击"开始选股"后、正式执行前

**弹窗内容：**
```
┌─────────────────────────────────────┐
│  🔍 选股参数确认          [✕]       │
├─────────────────────────────────────┤
│  当前策略：趋势突破                   │
│  适用场景：上升/震荡期                │
│                                     │
│  筛选条件：                          │
│  ✓ MA20 > MA60（趋势向上）           │
│  ✓ MACD金叉                          │
│  ✓ 量比 > 1.3                        │
│  ✓ 流通市值 > 50亿                   │
│  ✓ 非ST/非停牌                       │
│                                     │
│  预估耗时：2-3分钟                    │
│                                     │
│  [取消]        [确认开始选股]         │
└─────────────────────────────────────┘
```

#### 3. 筛选进度可视化

**当前状态：** 静态文字"筛选进行中..."

**建议状态：**
```
┌─────────────────────────────────────┐
│  ⏳ 选股筛选进行中...                │
│  ████████░░░░░░░░░░ 45%              │
│                                     │
│  当前步骤：板块资金流向分析...        │
│  已处理：2,450 / 5,200 只            │
│  预计剩余：1分30秒                   │
└─────────────────────────────────────┘
```

**技术方案：**
- 后端增加SSE (Server-Sent Events) 或轮询接口返回进度
- 前端显示进度条+当前步骤+处理数量

### P1 - 功能增强

#### 4. 筛选漏斗可视化重构

**当前设计：**
```
全市场 5200 → 基础过滤 4800 → 趋势确认 1200 → 板块资金 800 → 800 只入选
```

**建议设计：**
```
全市场          基础过滤        趋势确认        板块资金        最终入选
 5200    →      4800    →      1200    →       800    →       80
  │              │              │              │             │
 ▼              ▼              ▼              ▼            ▼
[████████]    [███████░░]    [██░░░░░░]    [█░░░░░░░]   [█░░░░░░░]
 100%          92%            23%            15%          1.5%

淘汰：400只    淘汰：3600只   淘汰：400只   淘汰：720只
```

**实现要点：**
- 使用漏斗图或桑基图展示筛选流程
- 显示每步淘汰数量和淘汰原因
- 悬停显示该步骤的具体筛选条件

#### 5. 结果表格优化

**当前表格列：** 排名 | 股票 | 现价 | 涨幅 | 总分 | 评分明细 | 板块 | 市值 | 主力净流入 | 操作

**建议表格列：** 排名 | 股票 | 评分 | 趋势 | 资金 | 操作

**详情展开行：** 点击行展开显示完整数据

```
┌─────────────────────────────────────────────────────────────┐
│ #  │    股票    │  评分   │   趋势   │   资金   │   操作    │
├─────────────────────────────────────────────────────────────┤
│ 🔥 │ 中国重汽   │ 88分   │ 📈 +5.2% │ 💰 流入  │ 👀  买入  │
│    │ 000951.SZ  │ ████████│ MA20↑    │ 1.2亿    │           │
├─────────────────────────────────────────────────────────────┤
│ ⚡ │ 汇宇制药   │ 76分   │ 📈 +3.8% │ 💰 流入  │ 👀  买入  │
│    │ 688553.SH  │ ██████░░│ MACD金叉 │ 0.8亿    │           │
└─────────────────────────────────────────────────────────────┘
```

**评分可视化：**
- 使用进度条或星级展示评分
- 80分以上：金色/红色高亮
- 60-80分：蓝色正常
- 60分以下：灰色弱化

#### 6. 增加结果排序功能

**排序选项：**
- 智能排序（默认，综合评分+趋势强度）
- 评分从高到低
- 涨幅从高到低
- 市值从大到小/从小到大
- 主力净流入从高到低

**UI位置：** 表格右上角，下拉选择器

**当前空状态：**
```

暂无入选股票
```

**场景A：大盘下降期**
```

│              📉                     │
│      当前大盘处于下降阶段            │
│                                     │
│  系统建议观望，暂不推荐买入标的      │
│                                     │
│  [查看历史选股结果]  [强制测试筛选]  │
└─────────────────────────────────────┘
```

**场景B：筛选条件过严**
```
┌─────────────────────────────────────┐
│              🔍                     │
│      当前筛选条件下无匹配标的        │
│                                     │
│  建议：                             │
│  • 放宽市值限制（当前>100亿）        │
│  • 降低量比要求（当前>2.0）          │

│                                     │

└─────────────────────────────────────┘
```

#### 8. 快捷操作优化

**当前设计：** 每行两个按钮（👀 加入观察池、买入）

**建议设计：**
```

[👀] [💰] [📊] [⋮]

  - 加入观察池
  - 模拟买入
  - 查看详情
  - 加入对比
```
