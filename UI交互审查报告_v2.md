## 三、问题清单（按优先级排序）

### 🔴 P0 — 影响核心体验，优先修复

#### P0-1：内联 style 泛滥，维护性极差
- **问题**：模板中存在 **300+ 处内联 `style="..."`**，与 CSS 设计 token 完全脱节
- **典型案例**：
  ```html
  <!-- 当前 -->
  <div style="display:grid;grid-template-columns:repeat(4,1fr);gap:16px;margin-bottom:20px">
  <div style="text-align:center;padding:16px;background:var(--bg-panel);border-radius:12px">
  <div style="font-size:13px;color:var(--text-muted);margin-bottom:8px">
  
  <!-- 应该 -->
  <div class="regime-metrics-grid">
  <div class="metric-card metric-card--center">
  <div class="metric-card__label">
  ```
- **影响**：修改主题/字体/间距需要逐行翻找，全局样式更新无法生效
- **建议**：将高频复用的 inline style 提取为 utility class 或组件 class

---

#### P0-2：选股结果表格信息过载
- **问题**：单行展示 **10 列**（排名、股票、现价、涨幅、总分、评分明细、板块、市值、主力净流入、操作），导致：
  - 移动端横向滚动无法使用
  - 评分明细（4行数字）在 12px 小字体下难以辨读
  - 操作列按钮 `👀 买入` 字号 12px，点击区域不足 36px
- **建议**：
  ```
  精简主表：排名 | 股票 | 评分(进度条) | 涨幅 | 操作
  点击行展开：现价/板块/资金/评分明细
  ```

---

#### P0-3：Tab 导航在小屏幕下溢出
- **问题**：`tab-nav` 使用 `display:flex;gap:0`，没有 `flex-wrap:wrap` 或横向滚动
- 当前 5 个 Tab（📊持仓 🔍选股 📝复盘 🎯策略 👀自选），在手机端 < 480px 时按钮会被裁切
- **当前 CSS**（line 106）：无 overflow 处理
- **建议**：
  ```css
  .tab-nav {
    overflow-x: auto;
    -webkit-overflow-scrolling: touch;
    scrollbar-width: none; /* 隐藏滚动条 */
    scroll-snap-type: x mandatory;
  }
  .tab-btn { scroll-snap-align: start; flex-shrink: 0; }
  ```

---

#### P0-4：持仓表格操作列按钮过小
- **问题**：`卖/加/🔔/⋮` 四个按钮挤在一行，样式为 `padding:4px 8px`，实际点击区域约 28px，低于移动端最小 44px 要求
- **对比**：CSS 中 `.btn-xs` 定义 `min-height:36px`，但操作列按钮用了内联覆盖
- **建议**：移除内联 `style="font-size:12px;padding:4px 8px"`，还原 `.btn-xs` 标准高度，或提取为 `.btn-icon-only`

---

### 🟡 P1 — 体验问题，应尽快修复

#### P1-1：加载状态不一致
- **问题**：系统中存在 4 种不同的加载态：
  1. `⏳ 加载中...`（市场数据面板）
  2. `⏳ 分析市场数据中...`（策略中心）
  3. 旋转 spinner（选股按钮旁的小圆圈）
  4. 按钮 disabled + 文字变化（选股主按钮）
- **建议**：统一为两种：① 页面级 → 骨架屏 ② 按钮级 → spinner + disabled

#### P1-2：Toast 无队列管理
- **问题**：`app.js` 中 toast 靠 `setTimeout` 覆盖，快速触发多个操作时（批量加入观察池 → 刷新）旧 toast 会被直接替换
- **建议**：建立 toast 队列，或改为右下角叠加式通知栈

#### P1-3：弹窗关闭方式不统一
- **问题**：
  - 大部分弹窗：`@click.self` 点背景关闭 ✅
  - 部分弹窗（止损止盈）：没有背景关闭支持 ❌
  - 部分弹窗：✕ 按钮在标题右侧，但没有统一的关闭按钮 class
- **建议**：统一所有弹窗为：`ESC 关闭` + `背景遮罩点击关闭` + 右上角 ✕ 标准按钮

#### P1-4：选股空状态缺乏上下文引导
- **当前**：
  ```
  🤷
  当前筛选条件下无匹配标的（或：大盘下降期）
  ```
- **问题**：显示了原因，但没有给出下一步操作
- **建议**：参考 `选股模块交互优化建议.md` P2 方案，区分"大盘下降期"和"条件过严"两种场景，提供对应建议操作

#### P1-5：策略运行按钮状态反馈弱
- **当前**：运行中时按钮变为 `⏳ 筛选进行中...`，旁边有一行黄色文字 `预计需要3-5分钟，请耐心等待...`
- **问题**：没有进度感，用户不知道到底执行到哪一步了
- **建议**：增加后端 SSE 或轮询进度端点（每5秒返回当前步骤+处理数量），前端显示：
  ```
  ████████░░░░ 60% | 正在分析板块资金流向... (已处理 3,200 / 5,400 只)
  ```

#### P1-6：「市场」Tab 被隐藏但功能已完备
- **问题**：代码中市场 Tab 完整实现了涨停/北向/龙虎榜/板块资金/板块轮动，但被注释隐藏
- **影响**：功能写了但用户用不到，是浪费
- **建议**：评估数据可靠性后开放，或添加「🔧 实验性功能」标签放开

#### P1-7：Header 操作区按钮过多且无层级
- **当前**：头部右侧有 `字体切换 + 用户信息 + 退出 + 导入 + 导出 + 刷新` 6 个控件
- **问题**：在 1200px 以下的宽度显示拥挤，移动端直接溢出
- **建议**：
  ```
  主要操作：🔄 刷新（保留）
  折叠到菜单：导入/导出/字体设置
  用户区：头像 + 下拉菜单（含退出/设置）
  ```

### 🔵 P2 — 体验提升，逐步优化

#### P2-1：没有深色/浅色模式切换
- 当前只有深色主题，没有跟随系统 `prefers-color-scheme` 的切换
- 对比 `系统全面审查报告.md` UI-09 建议：增加主题切换支持

#### P2-2：大盘指数卡片高低点结构标签信息孤立
- `↗ 上升 / ↘ 下降 / → 震荡` 的结构标签在指数卡中，但选股策略推荐没有直接链接到这个判断
- 建议：在策略推荐理由中明确引用当前指数结构（"当前上证处于↗上升，推荐趋势突破策略"）

#### P2-3：持仓表格止损/止盈列显示简陋
- 当前：`损28.5/盈35.0` 纯文字
- 建议：用小色块条显示当前价相对止损/止盈的位置（类似进度条），一眼能看出风险位置

#### P2-4：数字格式化不一致
- 问题：
  - 有些地方：`¥12,345.67`（formatNum）
  - 有些地方：`¥12345.67`（直接 toFixed）
  - 主力净流入：`123万`（formatAmount）
  - 市值：`123.4亿`（circ_mv_yi 字段）
- 建议：统一一套 `formatMoney(val, unit='auto')` 函数

#### P2-5：复盘报告的自定义列布局在不同屏幕下错乱
- **问题**：逐笔交易明细使用硬编码 grid：
  ```html
  grid-template-columns:50px 90px 100px 1fr 100px 90px 100px 80px
  ```
  在 768px 以下屏幕宽度下，合计列宽 620px > 屏幕宽，但没有 `overflow-x:auto` 包裹

#### P2-6：观察池卡片信息密度低
- **当前**：一张卡片展示：名称/代码/现价/涨幅/策略/加入时间/操作按钮
- **问题**：同时展示 2 个操作按钮（📋选股策略建议 + K线图 + 移除），在小卡片内显得挤
- **建议**：参考持仓操作列的 `⋮` 下拉菜单模式，收纳次要操作

#### P2-7：回测参数选择器样式与全局设计脱节
- 两个 `<select>` 使用了内联样式，而非 `.form-group select` 标准样式
- 视觉上偏小（padding:5px 8px），与其他控件 44px 最小高度不一致

#### P2-8：筛选漏斗缺少视觉权重
- 当前：水平 `pill` 标签 + `→` 箭头，纯文字，缺乏数字对比感
- 建议：每个步骤增加 minibar（宽度按比例），直观显示淘汰率

## 四、CSS 代码质量问题

### 内联样式占比分析

通过统计 `index.html`，发现内联 `style` 属性出现约 **350 次**，占所有行的 **35%**。主要集中在：

| 场景 | 内联 style 数量 | 建议 |
|------|----------------|------|
| 策略中心 Tab | ~80 处 | 提取为 `.regime-card / .regime-grid` |
| 复盘报告 Tab | ~70 处 | 提取为 `.review-trade-row / .review-summary` |
| 市场数据 Tab | ~60 处 | 提取为 `.market-list-row / .market-metric` |
| 弹窗内容 | ~50 处 | 提取为 `.modal-body-row / .advice-section` |

### var(--bg-panel) 未定义
- **问题**：CSS token 中定义了 `--bg-primary/secondary/card/hover`，但模板中多处使用了 `var(--bg-panel)`，该变量未在 `:root` 或 `[data-theme]` 中定义
- **影响**：这些区域回退到浏览器默认（白色），在深色主题下显示为白色背景块
- **位置**：策略中心 `background:var(--bg-panel)`、复盘报告 `background:var(--bg-panel)` 约 15 处

### 响应式缺口
```css
/* 当前 Media Query 只有两个断点 */
@media(max-width:768px) { ... }
@media(max-width:480px) { ... }

/* 缺少的断点 */
@media(max-width:1024px) { /* 平板端：双列 → 单列 */ }
@media(max-width:375px)  { /* iPhone SE 等超小屏 */ }
```

## 七、Quick Wins（今晚可完成）

以下 5 个问题修复工时 < 30 分钟，收益明显：

```css
/* Fix 1: 定义缺失的 --bg-panel 变量 */
html {
    --bg-panel: #141824; /* 等同于 --bg-secondary */
}

/* Fix 2: Tab 导航溢出修复 */
.tab-nav {
    overflow-x: auto;
    -webkit-overflow-scrolling: touch;
    scrollbar-width: none;
}
.tab-nav::-webkit-scrollbar { display: none; }
.tab-btn { flex-shrink: 0; }

/* Fix 3: 复盘明细表格横向滚动 */
.review-trades-wrapper {
    overflow-x: auto;
}

/* Fix 4: 操作按钮最小高度恢复 */
.actions .btn-xs {
    min-height: 36px !important;
    padding: 8px 12px !important;
}

/* Fix 5: 平板端 1024px 断点补充 */
@media(max-width:1024px) {
    .charts-grid { grid-template-columns: 1fr; }
    .param-grid { grid-template-columns: 1fr; }
}
```

## 八、结论

**系统整体视觉质量在同类个人工具中处于中上水平**，深色主题、设计 token 体系、卡片式策略选择器都是值得保留的优秀设计。

核心待解决的矛盾是：**CSS 设计系统建立得很好，但模板执行时大量绕过了这个系统（内联 style）**，导致了一致性和可维护性的双重下降。

建议优先从"Quick Wins"入手，再按阶段推进。第一阶段不需要大改，修几个 CSS 变量和 Media Query 就能明显改善移动端体验。
